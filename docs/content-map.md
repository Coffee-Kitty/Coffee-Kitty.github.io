# 内容分类

这个博客建议按“公开展示”和“个人沉淀”分层维护。公开层负责给别人看，也方便自己回顾；个人层保留原始记录、草稿和素材，不进入导航和时间线。

## 公开展示

| 分类 | 放什么 | 当前目录 |
| --- | --- | --- |
| 研究和论文 | 论文阅读、研究计划、实验想法、SWE/Agent 方向材料 | `/paper/` |
| LLM / Agent 工程 | 模型部署、SFT、Agent 框架、工具链和实践记录 | `/llm/` |
| 工程工具箱 | Python、Linux、VS Code、Playwright、设计模式、调试经验 | `/code/` |
| 算法和基础 | 传统算法、深度学习、程序分析、分布式系统等系统学习内容 | `/传统算法学习/`、`/深度学习/`、其他课程目录 |
| 阶段复盘 | 年末回顾、公开学习总结、重要选择和方法论 | `/life/` 中筛选后的文章 |

## 个人沉淀

| 分类 | 处理方式 | 原因 |
| --- | --- | --- |
| 日记 | 不放入首页、侧边栏和公开时间线 | 太私人，适合自查，不适合公开索引 |
| 简历、申请、密钥 | 不放入 `docs/`，不进入公开仓库 | 这类内容即使没有链接，也可能被访问或从仓库看到 |
| 原始素材 | 能压缩就压缩，能外链就外链 | 图片、PPT、ZIP 会拖慢仓库和页面加载 |
| 临时文件 | 加入 `.gitignore` | 避免 `.DS_Store`、Office 临时文件污染历史 |

## 建议的长期目录

```text
docs/
  README.md              # 首页
  content-map.md         # 内容分类
  learning-map.md        # 学习主线
  timeline.md            # 公开学习时间线，自动生成
  paper/                 # 研究和论文
  llm/                   # LLM / Agent 工程
  code/                  # 工程工具箱
  传统算法学习/           # 算法训练
  深度学习/               # 深度学习基础
  life/                  # 只放愿意公开的复盘和专题

private/                 # 不提交，放真正私人的日记、简历、密钥和草稿
```

## 新内容放哪里

| 新内容类型 | 建议位置 |
| --- | --- |
| 论文阅读 | `/paper/论文名.md` |
| 实验过程和结论 | `/paper/` 或 `/llm/`，看主题归属 |
| 部署和环境踩坑 | `/llm/` 或 `/code/` |
| 算法专题总结 | `/传统算法学习/` |
| 生活公开复盘 | `/life/` |
| 私人日记 | `private/diary/`，不要放进 `docs/` |
