# 算法学习

这个目录整合为三类：Hot 100、灵茶山艾府题单总览、按题单拆分的专题笔记。后续学习时优先更新对应专题文件，阶段性计划和进度写回总览。

## 总方法论

数据结构和算法不要只按名字记。更稳定的方式是从问题里的信息流出发：

```text
暴力原型
-> 重复信息
-> 信息形状
-> 数据结构/算法
-> 不变量
-> 操作模板
-> 代表题
```

每道题先问：

```text
1. 如果暴力做，我在枚举什么？
2. 暴力里哪些信息被重复计算？
3. 这些信息的形状是什么？线性、区间、集合关系、路径、优先级候选、状态转移，还是图？
4. 我需要维护什么不变量？
5. 这个结构如何支持查询和更新？
```

常见信息形状和工具：

| 信息形状 | 典型问题 | 常用工具 |
| --- | --- | --- |
| 线性顺序 | 最近、最早、窗口、单侧边界 | 栈、队列、单调栈、单调队列、双指针 |
| 连续区间 | 区间和、区间最值、区间修改、动态查询 | 前缀和、差分、ST 表、树状数组、线段树 |
| 集合关系 | 同组判断、连通块、关系合并、传递性 | 并查集、带权并查集 |
| 路径前缀 | 字符串前缀、二进制最优匹配、十进制字典序 | Trie、异或 Trie、十进制 Trie |
| 动态候选 | 当前最优、Top K、多路合并、中位数 | 堆、双堆、懒删除堆 |
| 图结构 | 可达性、最短路、拓扑依赖、环 | DFS、BFS、Dijkstra、拓扑排序、并查集 |
| 状态转移 | 子问题复用、选择序列、最优值 | 动态规划、记忆化搜索 |

真正要记住的不是模板本身，而是：

```text
这个结构维护了什么不变量？
为什么这个不变量足以回答问题？
每次更新后，不变量如何继续成立？
```

例如：

```text
前缀和：区间和 = 两个前缀状态相减。
单调队列：队列中下标在窗口内，值保持单调，队头就是窗口最值。
堆：堆顶是当前候选集合的最优者。
Trie：根到节点是一段前缀路径。
并查集：find(x) 是 x 所在集合的代表元。
树状数组：tree[i] 是以 i 结尾、长度 lowbit(i) 的区间信息。
```

一句话：

> 数据结构不是工具清单，而是为某种信息形状维护不变量的方法。

## 题单统一视角

算法题单可以统一成一句话：

> 算法的本质，是找到一种方式，不断复用题目中已经存在的联系。

更具体地说，绝大多数题都可以按这条链路思考：

```text
暴力枚举所有可能
-> 发现可复用的联系
-> 判断联系的形状
-> 选择维护方式
-> 建立不变量
-> 查询、更新、得到答案
```

所以结构本身就是算法的一部分。结构一旦确定，对应的查询、更新和正确性证明通常也基本确定。

| 专题 | 可复用的联系 | 常见维护方式 |
| --- | --- | --- |
| 前缀和 / 差分 | 区间信息可以由端点状态推出 | 保存累计状态或变化边界 |
| 滑动窗口 / 双指针 | 相邻窗口只差少量元素 | 维护窗口状态并移动端点 |
| 栈 / 队列 / 堆 | 候选元素有顺序、时间或优先级关系 | 维护候选池 |
| 单调栈 / 单调队列 | 一些候选永远不可能再成为答案 | 删除劣质候选，保留有用边界 |
| 二分算法 | 答案空间或判定结果存在单调性 | 用 check 复用可行性判断 |
| 贪心 | 某个局部选择可以安全固定 | 用交换、反悔或排序证明选择安全 |
| 动态规划 | 大问题由重复子问题组成 | 保存状态，复用转移结果 |
| 图论算法 | 点之间通过边、路径、连通性产生关系 | DFS、BFS、最短路、拓扑、生成树 |
| 并查集 | 同组关系具有传递性 | 合并集合，查询代表元 |
| Trie / 字符串 | 前缀、匹配过程或字符路径可以共享 | 建树、自动机、哈希或匹配函数 |
| 树状数组 / 线段树 / ST 表 | 区间信息可以被拆分再合并 | 维护区间块、树形分解或倍增块 |
| 位运算 | 二进制位之间存在独立性或恒等式 | 按位拆分、试填、状态压缩 |
| 数学算法 | 题目背后有公式、不变量、周期或计数关系 | 数论、组合、概率、几何、博弈 |
| 回溯 / 搜索 | 决策树路径上的状态可以增量维护 | 选择、递归、撤销、剪枝 |

遇到新题时，不要先问“这题套哪个模板”，而是先问：

```text
这道题里，什么关系可以被复用？
这个关系是线性的、区间的、集合的、路径的、状态的，还是图上的？
我需要维护的最小不变量是什么？
```

## 推荐学习顺序

学习顺序按三个标准排：依赖关系优先、复用频率优先、抽象难度递增。

| 阶段 | 主线 | 建议内容 | 为什么放这里 |
| --- | --- | --- | --- |
| 1 | 刷题方法和枚举基本功 | 如何科学刷题、复杂度、哈希表、排序、基础枚举 | 先建立“暴力原型”和复杂度意识 |
| 2 | 线性与区间基础 | 前缀和、差分、滑动窗口、双指针 | 高频，容易从暴力优化中自然推出 |
| 3 | 候选池结构 | 栈、队列、单调栈、单调队列、堆 | 训练“哪些候选还能留下”的直觉 |
| 4 | 基础搜索结构 | 网格图 DFS/BFS、树 DFS/BFS、回溯、链表基础 | 先掌握状态遍历，再进入复杂图论 |
| 5 | 关系和前缀结构 | 并查集、带权并查集、Trie、异或 Trie | 把“同组关系”和“路径共享”固化成结构 |
| 6 | 二分与贪心 | 二分答案、最小化最大值、最大化最小值、区间贪心、反悔贪心 | 开始训练证明：单调性和选择安全性 |
| 7 | 动态区间结构 | ST 表、树状数组、线段树 | 从静态区间到动态区间，衔接数据结构进阶 |
| 8 | 动态规划主线 | 入门 DP、背包、划分、状态机、区间 DP、树形 DP | DP 抽象度高，放在有足够题感之后 |
| 9 | 图论进阶 | 拓扑排序、基环树、最短路、最小生成树、网络流 | 需要搜索、堆、并查集等前置直觉 |
| 10 | 字符串进阶 | KMP、Z 函数、Manacher、字符串哈希、AC 自动机、后缀数组 | 模型独立且细节多，适合单独集中突破 |
| 11 | 位运算、数学、构造 | 位运算性质、数论、组合、概率、博弈、计算几何、构造题 | 更依赖题感和抽象转换，适合穿插复习 |

当前最推荐的主线是：

```text
常用数据结构
-> 二分算法
-> 贪心与思维
-> 动态规划
-> 图论算法
-> 字符串进阶
-> 数学、位运算、构造专题穿插巩固
```

其中常用数据结构内部可以按这个顺序推进：

```text
前缀和 / 差分
-> 栈 / 队列 / 单调结构
-> 堆
-> Trie
-> 并查集
-> ST 表
-> 树状数组
-> 线段树
```
