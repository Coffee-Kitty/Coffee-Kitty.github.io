## 母问题视角

这些数据结构不要只按名字记，更应该按“它在回答什么母问题”来记。

| 母问题 | 对应结构 | 直觉 |
| --- | --- | --- |
| 最近是谁？ | 栈 | 最近未处理元素最重要 |
| 最早是谁？ | 队列 | 最早进入的元素最早处理/失效 |
| 最优是谁？ | 堆 | 候选集合变化时，快速拿最大/最小/最高优先级 |
| 是否同组？ | 并查集 | 维护关系合并和连通块 |
| 是否有这个前缀路径？ | Trie | 按字符/二进制位/十进制位走路径 |
| 静态区间和是多少？ | 前缀和 | 区间和变成两个前缀状态相减 |

| 动态区间信息是什么？ | 线段树 | 区间查询 + 区间修改 |

## 候选池视角：栈、队列、堆的本质区别

很多数据结构都可以看成一个“候选池”：

```text
新元素进入候选池；
需要处理元素时，按照某种规则取出一个候选。
```

区别在于“谁有资格先出来”。

| 结构 | 谁先出来 | 核心直觉 | 适合问题 |
| --- | --- | --- | --- |
| 栈 | 最后进去的先出来 | 最近的最重要 | 括号匹配、表达式、邻项消除、DFS |
| 队列 | 最早进去的先出来 | 时间顺序最重要 | BFS、窗口过期、请求流 |
| 堆 | 优先级最高的先出来 | 最值/优先级最重要 | Top K、多路合并、任务调度 |
| 单调栈 | 最近且满足单调边界的候选先出来 | 最近未解决候选 | 下一个更大/更小、左右边界 |
| 单调队列 | 窗口内最优候选在队头 | 时间窗口 + 最值 | 滑动窗口最大/最小 |

最核心的区别：

```text
栈：我关心最近那个。
队列：我关心最早那个。
堆：我关心最优那个。
```

单调结构不是新的底层结构，而是在栈/队列基础上提前删除无用候选：

```text
单调栈 = 栈 + 删除不可能成为边界答案的候选。
单调队列 = 队列 + 删除不可能成为窗口最值的候选。
```

这个视角可以帮助判断该用什么结构：

- 如果当前元素只和最近未处理元素发生关系，优先想栈。
- 如果元素按时间进入、按时间过期，优先想队列。
- 如果只关心当前最大/最小/优先级最高的候选，优先想堆。
- 如果要找单侧最近更大/更小边界，优先想单调栈。
- 如果要维护滑动窗口内最大/最小值，优先想单调队列。

## 顺序结构对比：栈、单调栈、队列、单调队列

这四个结构都在处理“顺序”。区别在于：普通结构只维护顺序，单调结构还维护候选最优性。

最短理解：

```text
栈：先进后出，处理最近未处理元素，常见于匹配、抵消、嵌套、还原过程。
队列：先进先出，处理最早进入元素，常见于窗口过期、BFS、按时间顺序处理。
单调栈：先进后出 + 单调性，处理单侧最近边界，比如下一个更大/更小。
单调队列：先进先出 + 单调性，处理滑动窗口内最值，比如窗口最大/最小。
```

更准确地说，单调栈不是求“相邻最值”，而是求“右侧/左侧第一个满足条件的边界”。这个边界在原数组里不一定相邻，但在“未解决候选集合”里是相邻的。

| 结构 | 维护对象 | 核心顺序 | 弹出/删除的含义 | 典型问题 | 一句话模型 |
| --- | --- | --- | --- | --- | --- |
| 栈 | 最近未处理元素 | 后进先出 | 栈顶被匹配、抵消、结算 | 括号匹配、栈序列、邻项消除、表达式 | 最近未匹配/未解决的元素先处理 |
| 单调栈 | 未找到边界的候选元素 | 后进先出 + 值单调 | 栈顶被当前元素解决，答案可结算 | 下一个更大/更小、矩形面积、贡献法 | 最近未解决候选先被当前元素检验 |
| 队列 | 当前有效窗口/层级 | 先进先出 | 队头过期或本层处理完成 | 时间窗口、BFS、层序遍历 | 最早进入的元素最早失效 |
| 单调队列 | 当前窗口内的最优候选 | 先进先出 + 值单调 | 队头过期；队尾被新元素支配 | 滑动窗口最大值、受限窗口最值 | 既删过期元素，也删不可能成为最优的元素 |

### 普通栈：最近未处理

栈强调“最近”。如果当前元素只可能和最近的未处理元素发生关系，就优先想栈。

```text
括号匹配：右括号只可能匹配最近的左括号
小行星碰撞：当前负数只可能先撞最近的正数
验证栈序列：栈顶必须先出
```

### 单调栈：最近未解决候选

单调栈不是原数组里的真实相邻，而是“删掉已解决/无用元素后的候选相邻”。

```text
栈里保存还没找到答案的候选元素。
新元素来了，如果能解决栈顶，就不断弹栈结算。
如果不能解决栈顶，由于栈有单调性，也不能解决更深处的元素，于是停止。
```

通用动作：

```python
for i, x in enumerate(nums):
    while st and x 能解决 st[-1]:
        j = st.pop()
        结算 j 的答案
    st.append(i)
```

### 普通队列：最早进入，最早失效

队列强调“时间顺序”。如果元素按时间进入，过期也一定从最早进入的元素开始，就优先想队列。

```text
最近请求次数：最早请求最早离开窗口
BFS：先到达的状态先扩展
层序遍历：当前层从队头依次出队
```

### 单调队列：窗口合法性 + 候选最优性

单调队列比普通队列多维护一个性质：队列里的元素不仅在窗口内，而且按值保持单调。

```text
队头：当前窗口最优值
队尾：新元素进入时，被支配的旧候选从这里删除
```

通用动作：

```python
for i, x in enumerate(nums):
    while q and q[0] 已过期:
        q.popleft()
    while q and x 支配 q[-1]:
        q.pop()
    q.append(i)
    使用 q[0] 更新答案
```

### 阶段 1：枚举技巧

目标：看到双变量、多变量题时，先尝试固定一个变量，再用哈希表、最值、前后缀数组维护其它变量的信息。

引导问题：

- 当前元素应该先参与答案，还是先加入维护结构？

### 阶段 2：前缀和

目标：把区间和统一写成两个前缀和相减，再扩展到哈希、距离和、二维矩形。

引导问题：

- 区间 `[l, r]` 对应的是 `pre[r + 1] - pre[l]` 还是 `pre[r] - pre[l - 1]`？
- 哈希表里存的是前缀和的值、下标，还是出现次数？
- 绝对值距离和能不能借助有序性拆成左右两部分？

### 阶段 3：差分

目标：把多次区间修改压缩为边界变化，再用前缀和还原结果。

引导问题：

- 这题是多次区间查询，还是多次区间修改？
- 区间 `[l, r]` 加 `x` 时，`r + 1` 是否存在？
- 最终要还原数组，还是只需要边扫描边判断条件？

### 阶段 4：普通栈

目标：理解栈维护的是“最近还没被处理掉的元素”。

引导问题：

- 栈里保存的是值、下标、符号，还是一个状态？
- 当前元素会不会和栈顶发生匹配、抵消或合并？
- 遇到能弹栈的情况，为什么可以立刻弹？

### 阶段 5：单调栈

目标：把普通栈升级成“有单调性的候选集合”，快速找到左右边界。

引导问题：

- 题目要找的是左/右第一个更大，还是左/右第一个更小？
- 栈里应该存值还是下标？
- 弹栈时是在给被弹元素结算答案，还是在为当前元素找边界？
- 相等元素应该弹掉还是保留？这会影响贡献法里的去重边界。

### 阶段 6：队列和单调队列

目标：先用队列维护合法窗口，再用单调队列维护窗口最优值。

引导问题：

- 窗口为什么只需要从队头删除过期元素？

- 新元素能否支配队尾元素？如果能，为什么队尾可以删除？

目标：维护动态 Top K、动态中位数和带过期元素的最值。

- 堆里的元素会不会过期？如果会，怎么懒删除？

## 枚举

### 双变量 枚举右 维护左

​	对于 **双变量问题**，例如两数之和 $a_i+a_j=t$，可以枚举右边的 $a_j$，转换成 **单变量问题**，也就是在 $a_j$ 左边查找是否有 $a_i = t-a_j$，这可以用哈希表维护。
我把这个技巧叫做 **枚举右，维护左**。

​	

[1. 两数之和 - 力扣（LeetCode）](https://leetcode.cn/problems/two-sum/description/)

```python
class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:

        st = dict()
        for j in range(len(nums)):

            i = st.get(target-nums[j], -1)
            if i!=-1:
                return [i,j]

            st[nums[j] ]= j
        return [-1,-1]
```

### 枚举右 维护左侧最优值

[121. 买卖股票的最佳时机](https://leetcode.cn/problems/best-time-to-buy-and-sell-stock/)

题目可以看成求：

```text
max(prices[j] - prices[i])，其中 i < j
```

枚举卖出日 `j`，维护左侧 `0...j-1` 的最低买入价 `mn`。遍历到 `j` 时，先用旧的 `mn` 更新答案，再把 `prices[j]` 加入维护范围。

关键不变量：

```text
处理第 j 天的答案前，mn 是 prices[0...j-1] 的最小值。
```

```python
class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        ans = 0
        mn = prices[0]

        for j in range(1, len(prices)):
            ans = max(ans, prices[j] - mn)
            mn = min(mn, prices[j])

        return ans
```

补充：这题允许收益为 0，所以有些写法先更新 `mn` 再算答案也能通过。但从不变量上，先算答案更清楚，因为它严格保证买入日位于卖出日前面。

### 多变量 枚举中间

​	对于三个 或 四个变量的问题， 枚举中间的变量往往更好算

[2909. 元素和最小的山形三元组 II](https://leetcode.cn/problems/minimum-sum-of-mountain-triplets-ii/)

```python
class Solution:
    def minimumSum(self, nums: List[int]) -> int:
        # 对每个位置j，需要其左右最小元素
        # 然后计算各个位置j的最小元素和即可
        n = len(nums)
        pre, suf = [0] * n, [0] * n 
        
        pre_min = nums[0]
        for i in range(1,n):
            pre[i]=pre_min
            pre_min = pre_min if pre_min < nums[i] else nums[i]

        suf_min = nums[n-1]
        for j in range(n-2,-1,-1):
            suf[j] = suf_min
            suf_min = suf_min if suf_min < nums[j] else nums[j]
        # print(pre)
        # print(suf)
        res = -1
        for t in range(1,n-1):
            if nums[t] > pre[t] and  nums[t] > suf[t]:
                tmp = pre[t] + nums[t] + suf[t] 
                res = tmp if res == -1 or res > tmp else res 
        return res
```

## 前缀和、差分

### 基本前缀和



**左闭右开公式**：下标为左闭右开区间 $[\textit{left},\textit{right})$ 的元素和就是 $\textit{sum}[\textit{right}] - \textit{sum}[\textit{left}]$。

[303. 区域和检索 - 数组不可变](https://leetcode.cn/problems/range-sum-query-immutable/)

```python
class NumArray:

    def __init__(self, nums: List[int]):
        self.s = list( accumulate(nums, initial=0) )
        # 注意加了 initial后，从initial开始
        # [initial, initail+nums[0], initial+nums[1], ...]
        # [l,r]的元素和 也从 s[r]-s[l-1] 变成 s[r+1]-s[l]
		# 从而避免[0,r]的特判
        
    def sumRange(self, left: int, right: int) -> int:
        return self.s[right+1]-self.s[left]


```



### 前缀和与哈希

[560. 和为 K 的子数组](https://leetcode.cn/problems/subarray-sum-equals-k/)

这题的关键不是“前缀和”本身，而是把区间和改写成：

```text
pre[j + 1] - pre[i] = k
```

所以当我们枚举右端点 `j` 时，只需要去左侧已经出现过的前缀和里找：

```text
pre[j + 1] - k
```

这里哈希表里存的是“前缀和出现了多少次”，不是只存有没有出现。因为同一个前缀和可能对应多个左端点。

`0:1` 的含义是“空前缀”出现了 1 次。它专门用来处理从下标 0 开始的子数组。

例如 `nums = [3, 1, 2]`，`k = 3`：

- 当枚举到 `j = 0` 时，当前前缀和是 `pre[1] = 3`
- 需要找的是 `pre[1] - k = 0`
- 如果哈希表里有 `0:1`，就说明存在一个左边界在数组开头之前的虚拟位置
- 这正好对应子数组 `[0, 0]`

```python
class Solution:
    def subarraySum(self, nums: List[int], k: int) -> int:
        # 由 多个 子数组的和 可以联想到前缀和
        # 此外，假设nums[i, j]的和为k， 
        # 枚举j， 则有 pre[j]-pre[i-1] = k
        # 也就是对于枚举的j，看其左侧是否存在 pre[i-1] 使得上式成立
        # 时间复杂度应为 n*2，可以通过
        # 进一步的，借鉴 两数之和的思路，则变为在左侧哈希表中寻找 pre[j]-k
        # 则时间复杂度变为 n

        pre = list( accumulate(nums, initial=0 ))
        st = dict()
        res = 0

        # 空前缀：用来统计从下标 0 开始的子数组
        st[0] = 1

        for j in range(len(nums)): 
            cur = pre[j + 1]
            if cur - k in st:
                res += st[cur - k]
            
            st[cur] = st.get(cur, 0) + 1
        return res
```

题后总结：

- `st` 存的是前缀和出现次数：`st[x] = 前缀和 x 已经出现了几次`。
- `st[0] = 1` 表示空前缀出现 1 次，用来统计从下标 0 开始的子数组。
- 枚举右端点时，当前前缀和为 `cur`，需要找左侧是否有 `cur - k`。
- 如果 `cur - k` 出现了 `st[cur - k]` 次，就说明有这么多个左边界能和当前右端点组成和为 `k` 的子数组。
- 必须先查询，再加入当前 `cur`，保证哈希表里维护的是当前右端点左侧的前缀和。

[1685. 有序数组中差绝对值之和](https://leetcode.cn/problems/sum-of-absolute-differences-in-a-sorted-array/)

有序数组的距离和可以拆成左右两部分。对 `nums[i] = x`：

```text
左侧贡献 = x * i - sum(nums[0...i-1])
右侧贡献 = sum(nums[i+1...n-1]) - x * (n - i - 1)
```

如果使用左闭右开的前缀和 `pre`，则：

```text
左侧贡献 = x * i - pre[i]
右侧贡献 = pre[n] - pre[i + 1] - x * (n - i - 1)
```

这类题的核心条件是数组有序，所以左边元素都 `<= x`，右边元素都 `>= x`，绝对值可以被拆掉。

```python
class Solution:
    def getSumAbsoluteDifferences(self, nums: List[int]) -> List[int]:
        """
            1e5所以复杂度至少 n^2肯定不行

            对于每个元素i，都需要跟其他所有元素计算下绝对值差。那这样肯定是 n^2
            能否前缀和？或者是说用 整个元素组和 - 元素i*元素数组长度呢?不能，因为是绝对值， 发现位于元素i左侧的，应该是元素i-左侧元素， 右侧则为 -元素i
            而这样的左右两侧的和，恰好能用前缀和得到
            这样 on 即可解决
        """ 

        res = []
        pre = list(accumulate(nums,initial=0)) #i+1代表i
        for i,tmp in enumerate(nums):
            lef = tmp*i - pre[i]
            rig = pre[len(nums)]-pre[i+1] - tmp * (len(nums)-1-i)
            res.append(lef+rig)
        return  res

```



[2615. 等值距离和](https://leetcode.cn/problems/sum-of-distances/)

这题的关键是把相同值的下标收集起来。对每个值 `v`，它对应一组递增下标：

```text
idx = [i0, i1, i2, ...]
```

对同一组 `idx` 来说，问题就变成了上一题的距离和：对每个下标 `idx[p] = x`，计算它到组内其它所有下标的距离之和。

使用前缀和：

```text
左侧贡献 = x * p - pre[p]
右侧贡献 = pre[m] - pre[p + 1] - x * (m - p - 1)
```

注意答案要写回原数组下标 `x`，即 `ans[x] = 左侧贡献 + 右侧贡献`。

```python
class Solution:
    def distance(self, nums: List[int]) -> List[int]:
        # 1.收集起来
        # 2.按照距离和的思路即可

        d = defaultdict(list)
        pre = defaultdict(list)
        for i,num in enumerate(nums):
            d[num].append(i)
            if pre[num]:
                pre[num].append(i+pre[num][-1])
            else:
                pre[num].append(i)
        # print(d)
        # print(pre)

        arr = []
        for i,num in enumerate(nums):
            if len(d[num]) == 1:
                arr.append(0)
            else:
                j = bisect.bisect_left(d[num], i) # 到这里已经nlogn了

                # 所以这里用前缀和+二分优化
                lef = -pre[num][j-1] + j*i if j>=1 else 0
                rig = pre[num][-1]-pre[num][j]-(len(d[num])-j-1)*i
                arr.append(lef+rig)

        return arr
```

[304. 二维区域和检索 - 矩阵不可变](https://leetcode.cn/problems/range-sum-query-2d-immutable/)

定义：

```text
pre[i + 1][j + 1] 表示 matrix[0...i][0...j] 这个左上角矩形的和
```

构造：

```python
class NumMatrix:

    def __init__(self, matrix: List[List[int]]):
        m, n = len(matrix), len(matrix[0])
        self.pre = [[0] * (n + 1) for _ in range(m + 1)]

        for i in range(m):
            for j in range(n):
                self.pre[i + 1][j + 1] = (
                    self.pre[i][j + 1]
                    + self.pre[i + 1][j]
                    - self.pre[i][j]
                    + matrix[i][j]
                )

    def sumRegion(self, row1: int, col1: int, row2: int, col2: int) -> int:
        return (
            self.pre[row2 + 1][col2 + 1]
            - self.pre[row1][col2 + 1]
            - self.pre[row2 + 1][col1]
            + self.pre[row1][col1]
        )
```

核心理解：

```text
目标矩形 = 大矩形 - 上面多余部分 - 左边多余部分 + 左上角重复减掉的部分
```

## 差分

[1109. 航班预订统计](https://leetcode.cn/problems/corporate-flight-bookings/)

差分的核心是把区间加法转成两个边界变化：

```text
给 nums[l...r] 每个位置都加 x
等价于：
d[l] += x
d[r + 1] -= x
```

之后对差分数组 `d` 求前缀和，就能还原每个位置的最终值。

```python
class Solution:
    def corpFlightBookings(self, bookings: List[List[int]], n: int) -> List[int]:
        diff = [0] * (n + 1)

        for first, last, seats in bookings:
            l = first - 1
            r = last - 1
            diff[l] += seats
            diff[r + 1] -= seats

        ans = [0] * n
        cur = 0
        for i in range(n):
            cur += diff[i]
            ans[i] = cur

        return ans
```

注意：这里航班编号是从 1 开始的，所以要先转成 0-index。

[3355. 零数组变换 I](https://leetcode.cn/problems/zero-array-transformation-i/)

这题不要先想“到底每次减哪个位置”，而是换成容量视角：

```text
每个 query [l, r] 可以让区间内每个位置最多减 1。
所以对位置 i 来说，它最多能被减多少次 = 覆盖 i 的 query 数量。
```

因此只需要判断：

```text
cover[i] >= nums[i]
```

如果成立，就可以把 `nums[i]` 减到 0；如果某个位置覆盖次数不够，就一定不行。

用差分统计覆盖次数：

```python
class Solution:
    def isZeroArray(self, nums: List[int], queries: List[List[int]]) -> bool:
        n = len(nums)
        diff = [0] * (n + 1)

        for l, r in queries:
            diff[l] += 1
            diff[r + 1] -= 1

        cover = 0
        for i, x in enumerate(nums):
            cover += diff[i]
            if cover < x:
                return False

        return True
```

也可以用“负数差分”理解：每个区间贡献一次 `-1`，最终看 `nums[i] + dec[i] <= 0` 是否成立。但正数覆盖次数更清楚，因为它直接表达“最多能减几次”。

[2536. 子矩阵元素加 1](https://leetcode.cn/problems/increment-submatrices-by-one/)

二维差分是把矩形加法转成四个角的变化。对矩形：

```text
(row1, col1) 到 (row2, col2) 整体加 x
```

更新：

```text
diff[row1][col1] += x
diff[row2 + 1][col1] -= x
diff[row1][col2 + 1] -= x
diff[row2 + 1][col2 + 1] += x
```

右下角要 `+x`，因为它被下边界取消一次、又被右边界取消一次，重复取消了，需要补回来。

```python
class Solution:
    def rangeAddQueries(self, n: int, queries: List[List[int]]) -> List[List[int]]:
        diff = [[0] * (n + 1) for _ in range(n + 1)]

        for r1, c1, r2, c2 in queries:
            diff[r1][c1] += 1
            diff[r2 + 1][c1] -= 1
            diff[r1][c2 + 1] -= 1
            diff[r2 + 1][c2 + 1] += 1

        ans = [[0] * n for _ in range(n)]
        for i in range(n):
            for j in range(n):
                up = diff[i - 1][j] if i > 0 else 0
                left = diff[i][j - 1] if j > 0 else 0
                diag = diff[i - 1][j - 1] if i > 0 and j > 0 else 0
                diff[i][j] += up + left - diag
                ans[i][j] = diff[i][j]

        return ans
```

二维前缀和不要对每个格子都 `sum(sum(...))`，那会变成每个格子都重新扫一遍矩形。应该用递推：

```text
pre[i + 1][j + 1] =
    pre[i][j + 1] + pre[i + 1][j] - pre[i][j] + matrix[i][j]
```

[20. 有效的括号](https://leetcode.cn/problems/valid-parentheses/)

栈适合处理“最近的未匹配元素”。括号匹配中，栈里存尚未匹配的左括号；遇到右括号时，弹出栈顶左括号判断是否匹配。

```python
class Solution:
    def isValid(self, s: str) -> bool:
        st = []
        mp = {')': '(', ']': '[', '}': '{'}

        for ch in s:
            if ch in mp.values():
                st.append(ch)
            else:
                if not st or st[-1] != mp[ch]:
                    return False
                st.pop()

        return not st
```

关键点：

- 栈里存左括号。
- 右括号负责触发匹配。
- 最后栈必须为空，说明没有未匹配的左括号。

[946. 验证栈序列](https://leetcode.cn/problems/validate-stack-sequences/)

这题可以直接模拟真实栈。`pushed` 的入栈顺序固定，`popped` 表示期望的出栈顺序。

流程：

```text
依次把 pushed 里的元素入栈；
每入栈一个元素，就检查栈顶是否等于 popped[j]；
如果相等，就持续弹栈并移动 j；
最后如果所有元素都能按 popped 弹出，则合法。
```

```python
class Solution:
    def validateStackSequences(self, pushed: List[int], popped: List[int]) -> bool:
        st = []
        j = 0

        for x in pushed:
            st.append(x)
            while st and st[-1] == popped[j]:
                st.pop()
                j += 1

        return not st
```

为什么可以“能弹就弹”：

```text
如果当前栈顶已经等于 popped[j]，那么 popped[j] 只能由这个栈顶弹出。
延迟弹出不会创造新的可能性，反而可能挡住后续元素，所以直接弹出是安全的。
```

[735. 小行星碰撞](https://leetcode.cn/problems/asteroid-collision/)

栈可以处理邻项消除。这里栈里维护的是“已经处理过、内部不会再发生碰撞的小行星”。

关键不变量：

```text
栈内的小行星已经互不碰撞。
如果把负数看成向左、正数看成向右，那么稳定结构一定形如：负数段 + 正数段。
```

当前小行星 `x` 只有在下面条件同时满足时，才会和栈顶碰撞：

```text
st[-1] > 0 and x < 0
```

因为只有“左边向右，右边向左”才会相向而行。

```python
class Solution:
    def asteroidCollision(self, asteroids: List[int]) -> List[int]:
        st = []

        for x in asteroids:
            alive = True

            while alive and x < 0 and st and st[-1] > 0:
                if st[-1] < -x:
                    st.pop()
                    continue
                if st[-1] == -x:
                    st.pop()
                alive = False

            if alive:
                st.append(x)

        return st
```

三种碰撞结果：

- `st[-1] < -x`：栈顶爆炸，当前小行星继续向左，可能继续撞。
- `st[-1] == -x`：两个都爆炸，当前小行星消失。
- `st[-1] > -x`：当前小行星爆炸。

## 单调栈

> 对应灵茶山艾府题单：[单调栈](https://leetcode.cn/discuss/post/9oZFK9/)。

### 学习目标

- 能判断题目要找「左/右第一个更大或更小」。
- 能解释栈中元素为什么保持单调。
- 能处理贡献法、矩形面积、最小字典序等变形。

### 和普通栈的区别

普通栈强调“后进先出”，常用于括号匹配、模拟过程、邻项消除。

单调栈强调“维护一个单调的候选集合”。当新元素进入时，它会淘汰一批以后不可能再成为答案的元素。

核心问题：

```text
当前元素 x 会让栈顶元素失去价值吗？
如果会，就弹栈；如果不会，就入栈。
```

### 候选相邻

单调栈里的“相邻”不是原数组相邻，而是候选集合里的相邻。

以 `4, 2, 5` 为例：

```text
4 入栈，等待右侧更大元素
2 小于 4，也入栈，等待右侧更大元素
5 到来，先解决最近候选 2
2 被弹出后，4 变成新的最近候选
5 再解决 4
```

这说明单调栈维护的是：

```text
还没找到答案的候选元素。
```

新元素只需要从栈顶开始检查。因为栈有单调性，如果新元素不能解决栈顶，就更不能解决栈底方向那些更强的候选。

一句话：

```text
普通栈处理最近未匹配；
单调栈处理最近未解决候选。
```

### 最小字典序：最近决策可反悔

最小字典序也是单调栈的经典模型，但它和“下一个更大/更小”不完全一样。

它维护的不是某个元素的边界答案，而是：

```text
当前约束下，已经构造出来的最小字典序前缀。
```

字典序的关键是：

```text
越靠前的位置，影响越大。
```

所以如果当前字符更小，它有动力往前放。但它能不能往前放，不是看窗口里谁最大/最小，而是看“当前答案末尾那一段字符”能不能被反悔。

```text
当前字符想往前走；
它必须先越过答案末尾的字符；
末尾字符如果还能在后面补回来，或者删除次数还够，就可以弹掉；
弹掉一个末尾字符后，继续检查新的末尾字符。
```

这就是栈的原因：

```text
最近做出的决定，最先被推翻。
```

例如 `543` 中，`4` 到来时会先尝试替换最近放入的 `5`；`3` 到来时会先尝试替换最近放入的 `4`。这个过程是后进先出，不是先进先出。

和单调队列的区别：

```text
单调队列解决“旧元素过期”：最早进入窗口的元素最先失效。
最小字典序单调栈解决“最近决策可反悔”：答案末尾最近加入的元素最先被检查。
```

因此最小字典序模型的触发信号是：

- 从左到右构造一个答案序列。
- 新字符更小，可能让前缀更优。
- 已选字符能否删除，取决于“后面是否还能出现”“删除次数是否还够”“是否必须保留一次”等约束。
- 需要不断反悔答案末尾，而不是删除最早进入的元素。

代表题：

- [316. 去除重复字母](https://leetcode.cn/problems/remove-duplicate-letters/)：每个字符必须保留一次，所以只有当栈顶字符后面还会出现时，才能弹出。
- [402. 移掉 K 位数字](https://leetcode.cn/problems/remove-k-digits/)：必须删除 `k` 个数字，所以只有当 `k > 0` 时，才能弹出更大的栈顶数字。
- [1081. 不同字符的最小子序列](https://leetcode.cn/problems/smallest-subsequence-of-distinct-characters/)：和 316 本质相同。

一句话：

```text
普通单调栈：维护最近未解决的边界候选。
最小字典序单调栈：维护最近仍可反悔的答案前缀。
```

### 总模板

单调栈通常存下标，而不是只存值。原因是下标可以同时得到：

- 元素值：`nums[i]`
- 距离：`i - j`
- 区间长度：`right - left - 1`
- 答案写回位置：`ans[i]`

#### 下一个更大元素

维护单调递减栈。新元素更大时，说明它是栈顶元素右侧第一个更大值。

```python
st = []
ans = [-1] * n

for i, x in enumerate(nums):
    while st and nums[st[-1]] < x:
        j = st.pop()
        ans[j] = x
    st.append(i)
```

#### 下一个更小元素

维护单调递增栈。新元素更小时，说明它是栈顶元素右侧第一个更小值。

```python
st = []
ans = [-1] * n

for i, x in enumerate(nums):
    while st and nums[st[-1]] > x:
        j = st.pop()
        ans[j] = x
    st.append(i)
```

### 题型识别

可以优先想到单调栈的信号：

- 要找某个元素左边或右边第一个比它大/小的元素。
- 每个元素作为最小值或最大值，问它能影响多少区间。
- 矩形高度由当前柱子决定，需要向左右扩展边界。
- 要维护字典序，并且需要对答案末尾的最近选择进行反悔。

不适合单调栈的情况：

- 只需要全局最大/最小值，通常用变量或堆。
- 需要窗口内最大/最小值，通常用单调队列，因为那是“旧元素过期 + 窗口最优”。
- 需要任意区间查询，通常考虑线段树、树状数组或 Sparse Table。

### 关键不变量

#### 递减栈

```text
栈中下标对应的值从栈底到栈顶递减。
```

用途：找右侧第一个更大元素。

当 `nums[i] > nums[st[-1]]` 时，当前元素就是 `st[-1]` 右侧第一个更大值，因为中间更早出现的元素都没能把它弹掉。

#### 递增栈

```text
栈中下标对应的值从栈底到栈顶递增。
```

用途：找右侧第一个更小元素、柱状图边界、子数组最小值贡献。

### 相等元素怎么处理

相等元素是否弹栈不是固定答案，要看题型。

- 基础“下一个更大”：通常用 `<`，相等不算更大。
- 基础“下一个更小”：通常用 `>`，相等不算更小。
- 贡献法：必须一边严格、一边非严格，避免重复统计相同最小值。
- 字典序：通常用 `>` 弹出更大的字符，相等字符由 `visited` 控制。

## 队列

[933. 最近的请求次数](https://leetcode.cn/problems/number-of-recent-calls/)

队列适合维护按时间推进的窗口。每次新请求时间 `t` 到来后，队列中只保留 `[t - 3000, t]` 内的请求时间。

```python
class RecentCounter:

    def __init__(self):
        self.q = deque()

    def ping(self, t: int) -> int:
        self.q.append(t)

        while self.q[0] < t - 3000:
            self.q.popleft()

        return len(self.q)
```

关键不变量：

```text
每次 ping(t) 结束后，队列中的所有时间都在 [t - 3000, t] 内。
```

为什么只从队头删：请求时间按调用顺序递增，越早入队的越可能过期；如果队头没过期，后面的元素一定也没过期。

[239. 滑动窗口最大值](https://leetcode.cn/problems/sliding-window-maximum/)

单调队列同时维护两件事：

```text
1. 窗口合法性：队头下标必须在当前窗口内。
2. 单调性：队列中的值从队头到队尾单调递减。
```

支配关系：

```text
如果新来的 y 比队尾 x 更大或相等，并且 y 的下标更靠右，
那么 x 之后不可能再成为最大值，可以直接从队尾删除。
```

所以队列里存下标，而不是只存值。这样才能判断队头是否已经离开窗口。

```python
class Solution:
    def maxSlidingWindow(self, nums: List[int], k: int) -> List[int]:
        q = deque()
        ans = []

        for i, x in enumerate(nums):
            while q and nums[q[-1]] <= x:
                q.pop()
            q.append(i)

            if q[0] <= i - k:
                q.popleft()

            if i >= k - 1:
                ans.append(nums[q[0]])

        return ans
```

相等时可以弹旧元素，也可以保留旧元素。弹旧元素的好处是队列更短，并且保留的是更晚过期的新元素。

### [LCR 044. 在每个树行中找最大值](https://leetcode.cn/problems/hPov7L/)

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def largestValues(self, root: TreeNode) -> List[int]:
        if root is None:
            return []
        q = deque([])
        q.append(root)
        res = []
        while q:
            sz = len(q)
            tmp = -math.inf
            for i in range(sz):
                node = q.popleft()
                if node is not None:
                    tmp = max(node.val, tmp)
                    if node.left is not None:
                        q.append(node.left)
                    if node.right is not None:
                        q.append(node.right)
            res.append(tmp)
        return res
```

### [LCR 043. 完全二叉树插入器](https://leetcode.cn/problems/NaqhDT/)

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val

        self.candidate = deque()

        while q:
            node = q.popleft()
            if node.left:
                q.append(node.left)
            if node.right:
                q.append(node.right)
            if not (node.left and node.right):

            node.left = child
        else:
            node.right = child
            self.candidate.popleft()
        self.candidate.append(child)
        return node.val

    def get_root(self) -> TreeNode:
        return self.root
```

## 堆

堆的直觉是“动态最值”。当候选集合不断变化，但每次只关心当前最大/最小/优先级最高的元素时，可以优先考虑堆。

### [703. 数据流中的第 K 大元素](https://leetcode.cn/problems/kth-largest-element-in-a-stream/)

这题维护的不是全局最大值，而是：

```text
当前最大的 k 个数
```

在这个集合里，我们最关心的是“门槛”：

```text
前 k 大里面最小的那个 = 全局第 k 大
```

所以使用大小为 `k` 的小根堆：

```text
堆里保存当前前 k 大
堆顶是前 k 大里最小的
堆顶就是当前第 k 大
```

```python
class KthLargest:

    def __init__(self, k: int, nums: List[int]):
        self.k = k
        self.heap = nums
        heapify(self.heap)

        while len(self.heap) > k:
            heappop(self.heap)

    def add(self, val: int) -> int:
        heappush(self.heap, val)

        if len(self.heap) > self.k:
            heappop(self.heap)

        return self.heap[0]
```

题后总结：

- 堆适合动态维护最值。
- Top K 里维护的是一个大小为 `k` 的候选集合。
- 求前 `k` 大时，用小根堆维护门槛。
- 新数如果没资格进入前 `k` 大，会在 push 后被 pop 掉。

### [23. 合并 K 个升序链表](https://leetcode.cn/problems/merge-k-sorted-lists/)

多路合并也是堆的典型场景：

```text
候选集合不断变化，但每次只取当前最优候选。
```

每条链表内部已经升序，所以全局最小值一定在每条链表当前还没合并的头节点里。堆维护的是：

```text
每一路当前头节点
```

每次操作：

```text
1. 从小根堆弹出当前最小头节点
2. 接到答案链表后面
3. 如果这个节点还有 next，把 next 加入堆
```

```python
class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        heap = []

        for i, node in enumerate(lists):
            if node:
                heappush(heap, (node.val, i, node))

        dummy = ListNode()
        cur = dummy

        while heap:
            _, i, node = heappop(heap)
            cur.next = node
            cur = cur.next

            if node.next:
                heappush(heap, (node.next.val, i, node.next))

        return dummy.next
```

复杂度：

```text
设总节点数为 N，链表数量为 k。
每个节点入堆一次、出堆一次，堆大小最多为 k。
时间复杂度 O(N log k)，空间复杂度 O(k)。
```

注意：Python 的 `heapq` 比较元组时，如果前面的元素相等，会继续比较后面的元素。链表节点不能直接比较，所以元组里要放一个可比较的编号。

### [295. 数据流的中位数](https://leetcode.cn/problems/find-median-from-data-stream/)

中位数不是全局最值，而是动态有序序列的中间边界。

如果把所有数分成两半：

```text
left  = 较小一半
right = 较大一半
```

中位数只和两个边界有关：

```text
较小一半的最大值
较大一半的最小值
```

所以使用两个堆：

```text
left：大根堆，维护较小一半的最大值
right：小根堆，维护较大一半的最小值
```

Python 只有小根堆，因此 `left` 用负数模拟大根堆。

两个关键不变量：

```text
1. len(left) == len(right) 或 len(left) == len(right) + 1
2. left 中所有数 <= right 中所有数
```

```python
class MedianFinder:

    def __init__(self):
        self.left = []   # 大根堆，用负数保存较小一半
        self.right = []  # 小根堆，保存较大一半

    def addNum(self, num: int) -> None:
        heappush(self.left, -num)
        heappush(self.right, -heappop(self.left))

        if len(self.right) > len(self.left):
            heappush(self.left, -heappop(self.right))

    def findMedian(self) -> float:
        if len(self.left) > len(self.right):
            return -self.left[0]
        return (-self.left[0] + self.right[0]) / 2
```

直觉链条：

```text
堆适合动态最值；
中位数是动态序列的中间边界；
中间边界由左半边最大值和右半边最小值决定；
所以用两个堆维护两个动态边界。
```

和 Top K 的关系：

```text
Top K：一个堆维护一个门槛。
中位数：两个堆维护两个半区的边界。
```

### [480. 滑动窗口中位数](https://leetcode.cn/problems/sliding-window-median/)

这是 `295. 数据流的中位数` 的加强版：

```text
295：数据只进不出。
480：窗口滑动，数据既进也出。
```

中位数仍然用两个堆维护：

```text
small：较小一半，大根堆
large：较大一半，小根堆
```

难点在于删除。堆只能快速删除堆顶，不能快速删除堆中任意元素。但窗口滑动时，离开窗口的元素不一定在堆顶。

因此需要懒删除：

```text
用哈希表 delayed 标记已经过期但还没从堆里物理删除的元素。
每次需要使用堆顶前，先清理堆顶带标记的元素。
```

为什么可行：

```text
堆中间的过期元素暂时不会影响答案；
只有当它来到堆顶时，才可能影响答案；
所以等它到堆顶再删除。
```

代码结构：

```python
from heapq import heappush, heappop
from collections import defaultdict

class Solution:
    def medianSlidingWindow(self, nums: List[int], k: int) -> List[float]:
        small = []  # 大根堆，用负数保存较小一半
        large = []  # 小根堆，保存较大一半
        delayed = defaultdict(int)

        small_size = 0  # small 中有效元素数量
        large_size = 0  # large 中有效元素数量

        def small_top() -> int:
            return -small[0]

        def large_top() -> int:
            return large[0]

        def prune(heap):
            while heap:
                x = -heap[0] if heap is small else heap[0]
                if delayed[x] == 0:
                    break
                delayed[x] -= 1
                if delayed[x] == 0:
                    del delayed[x]
                heappop(heap)

        def make_balance():
            nonlocal small_size, large_size

            if small_size > large_size + 1:
                heappush(large, -heappop(small))
                small_size -= 1
                large_size += 1
                prune(small)
            elif small_size < large_size:
                heappush(small, -heappop(large))
                large_size -= 1
                small_size += 1
                prune(large)

        def add_num(num: int):
            nonlocal small_size, large_size

            if not small or num <= small_top():
                heappush(small, -num)
                small_size += 1
            else:
                heappush(large, num)
                large_size += 1

            make_balance()

        def erase_num(num: int):
            nonlocal small_size, large_size

            delayed[num] += 1

            if num <= small_top():
                small_size -= 1
                if num == small_top():
                    prune(small)
            else:
                large_size -= 1
                if large and num == large_top():
                    prune(large)

            make_balance()

        def get_median() -> float:
            if k % 2:
                return float(small_top())
            return (small_top() + large_top()) / 2

        ans = []

        for i in range(k):
            add_num(nums[i])
        ans.append(get_median())

        for i in range(k, len(nums)):
            add_num(nums[i])
            erase_num(nums[i - k])
            ans.append(get_median())

        return ans
```

题后总结：

- 懒删除堆针对的是堆的弱点：无法快速删除非堆顶元素。
- `delayed` 记录过期元素，操作堆顶前用 `prune` 清理。
- 堆里可能有过期元素，所以 `len(heap)` 不是有效大小，需要维护 `small_size` 和 `large_size`。
- 滑动窗口求最值也可以用堆加懒删除，但通常单调队列更简单，复杂度也更优。

### 堆和单调队列都能做滑动窗口最值吗

可以。比如滑动窗口最大值：

- 单调队列：维护窗口内递减候选，队头就是最大值，时间复杂度 `O(n)`。
- 堆：维护 `(-value, index)`，堆顶是最大值；如果堆顶下标过期，就弹出，时间复杂度 `O(n log n)`。

注意术语：

```text
优先队列的常见底层实现就是堆。
所以“用优先队列做滑动窗口最值”通常就是“用堆做”，复杂度仍然是 O(n log n)。
真正做到 O(n) 的是单调队列，不是优先队列。
```

因此：

```text
窗口最值：优先单调队列。
窗口中位数：单调队列不够，因为中位数不是最大/最小；用双堆。
动态 Top K / 多路合并 / 优先级模拟：优先堆。
```

## Trie 字典树

Trie 的直觉是“把字符串按字符拆成路径”。如果字符集是小写字母，每个节点最多有 26 个孩子。

核心价值：

```text
公共前缀共享同一段路径。
```

例如 `app` 和 `apple` 共享：

```text
a -> p -> p
```

节点上需要区分两件事：

```text
路径存在：说明有某些单词经过这里。
终点标记 is_end：说明这里正好是一个完整单词的结尾。
```

所以：

```text
startsWith("app") 只需要路径存在。
search("app") 还需要最后节点 is_end = True。
```

### [208. 实现 Trie 前缀树](https://leetcode.cn/problems/implement-trie-prefix-tree/)

数组写法：

```python
class Trie:

    def __init__(self):
        self.children = [None] * 26
        self.is_end = False

    def insert(self, word: str) -> None:
        node = self
        for ch in word:
            i = ord(ch) - ord('a')
            if node.children[i] is None:
                node.children[i] = Trie()
            node = node.children[i]
        node.is_end = True

    def search_prefix(self, prefix: str):
        node = self
        for ch in prefix:
            i = ord(ch) - ord('a')
            if node.children[i] is None:
                return None
            node = node.children[i]
        return node

    def search(self, word: str) -> bool:
        node = self.search_prefix(word)
        return node is not None and node.is_end

    def startsWith(self, prefix: str) -> bool:
        node = self.search_prefix(prefix)
        return node is not None
```

### Trie 复杂度直觉

Trie 的复杂度基本按“字符串长度”算，而不是按“已经插入了多少个单词”算。

原因是一次操作只会沿着目标字符串的字符路径走：

```text
insert("apple")：走 a -> p -> p -> l -> e
search("apple")：走 a -> p -> p -> l -> e
startsWith("app")：走 a -> p -> p
```

设字符串长度为 `L`：

```text
insert(word)：O(L)
search(word)：O(L)
startsWith(prefix)：O(L)
erase(word)：O(L)
```

建树复杂度：

```text
如果所有插入单词的总长度是 S，则建 Trie 的时间复杂度是 O(S)。
Trie 节点数最多是 S + 1，因为每个字符最多新建一个节点，再加根节点。
```

空间复杂度取决于孩子表示方式：

```text
数组 children[26]：每个节点固定 26 个指针，时间常数小，空间常数大。
字典 children：只存实际存在的边，空间更灵活，哈希查找平均 O(1)。
```

所以在算法题里通常写：

```text
时间复杂度：O(字符串长度)
空间复杂度：O(所有插入字符串的总长度)
```

1804 的 `countWordsStartingWith(prefix)` 也是这个思想的延伸：查询想要 `O(len(prefix))`，就不能走到前缀节点后 DFS 整棵子树，而要在插入/删除时提前维护 `prefix_count`。

### [1804. 实现 Trie II 前缀树](https://leetcode.cn/problems/implement-trie-ii-prefix-tree/)

这是 `208` 的计数版，需要支持：

```text
insert(word)
countWordsEqualTo(word)
countWordsStartingWith(prefix)
erase(word)
```

和 `208` 的区别：

```text
208：只关心单词是否存在。
1804：关心单词出现了几次，以及以某个前缀开头的单词有几个。
```

因此每个节点除了孩子，还要维护两个计数：

```text
prefix_count：有多少个单词经过这个节点
end_count：有多少个单词正好在这个节点结束
```

插入 `word`：

```text
沿路径每个节点 prefix_count += 1
最后一个节点 end_count += 1
```

删除 `word`：

```text
沿路径每个节点 prefix_count -= 1
最后一个节点 end_count -= 1
```

计数查询：

```text
countWordsEqualTo(word)：走到 word 的末尾，返回 end_count
countWordsStartingWith(prefix)：走到 prefix 的末尾，返回 prefix_count
```

代码骨架：

```python
class Trie:

    def __init__(self):
        self.children = {}
        self.prefix_count = 0
        self.end_count = 0

    def insert(self, word: str) -> None:
        node = self
        for ch in word:
            if ch not in node.children:
                node.children[ch] = Trie()
            node = node.children[ch]
            node.prefix_count += 1
        node.end_count += 1

    def _find(self, s: str):
        node = self
        for ch in s:
            if ch not in node.children:
                return None
            node = node.children[ch]
        return node

    def countWordsEqualTo(self, word: str) -> int:
        node = self._find(word)
        return 0 if node is None else node.end_count

    def countWordsStartingWith(self, prefix: str) -> int:
        node = self._find(prefix)
        return 0 if node is None else node.prefix_count

    def erase(self, word: str) -> None:
        node = self
        for ch in word:
            node = node.children[ch]
            node.prefix_count -= 1
        node.end_count -= 1
```

注意：这里 `prefix_count` 统计的是经过当前节点的单词数量，不包含根节点。题目保证 `erase(word)` 时 `word` 存在，所以删除时可以直接沿路径走。

```python
        return node is not None
```

### [421. 数组中两个数的最大异或值](https://leetcode.cn/problems/maximum-xor-of-two-numbers-in-an-array/)

异或 Trie 不是按字符建树，而是按二进制位建树。每个节点最多两个孩子：

```text
0 和 1
```

这题可以重新理解成“枚举右，维护左”：

```text
枚举当前数 nums[j]；
用 Trie 维护左侧已经出现过的 nums[0...j-1]；
查询 nums[j] 能和左侧哪个数异或出最大值；
再把 nums[j] 插入 Trie。
```

为什么先查再插：

```text
这样可以保证当前数只和左侧数字配对，不会和自己配对。
```

查询时从高位到低位贪心：

```text
当前位是 0，优先找 1；
当前位是 1，优先找 0。
```

因为异或中：

```text
相同位 = 0
不同位 = 1
```

并且高位权重更大，所以必须从高位开始。

以题目示例：

```text
nums = [3, 10, 5, 25, 2, 8]
```

当枚举到：

```text
25 = 11001
```

Trie 中已有：

```text
3  = 00011
10 = 01010
5  = 00101
```

为了让异或最大，`25` 希望找：

```text
25: 1 1 0 0 1
想: 0 0 1 1 0
```

最终能沿着 `5 = 00101` 的路径走，得到：

```text
25 ^ 5 = 11001 ^ 00101 = 11100 = 28
```

代码：

```python
class Node:
    def __init__(self):
        self.children = [None, None]


class Trie:
    def __init__(self):
        self.root = Node()

    def insert(self, x: int) -> None:
        node = self.root
        for i in range(30, -1, -1):
            b = (x >> i) & 1
            if node.children[b] is None:
                node.children[b] = Node()
            node = node.children[b]

    def query(self, x: int) -> int:
        node = self.root
        ans = 0
        for i in range(30, -1, -1):
            b = (x >> i) & 1
            want = b ^ 1

            if node.children[want]:
                ans |= 1 << i
                node = node.children[want]
            else:
                node = node.children[b]

        return ans


class Solution:
    def findMaximumXOR(self, nums: List[int]) -> int:
        trie = Trie()
        ans = 0

        trie.insert(nums[0])
        for x in nums[1:]:
            ans = max(ans, trie.query(x))
            trie.insert(x)

        return ans
```

题后总结：

- 异或 Trie 是按二进制位建的 Trie。
- 它和普通 Trie 的共同点是“按路径查询”，区别是路径字符变成了二进制位。
- 这题仍然可以套“枚举右，维护左”：Trie 维护左侧数字集合。
- 不建议一开始把所有数字都插进去再查询，因为这样会模糊“当前数和谁配对”的关系，也容易误把自己作为候选。
- 查询时从高位到低位优先走相反位，贪心最大化异或值。

### 特殊标记：十进制 Trie 与字典序

数字也可以按十进制位建立 Trie：

```text
12345 -> 1 -> 2 -> 3 -> 4 -> 5
```

每个节点最多 10 个孩子 `0-9`。这种思维常用于数字前缀和字典序问题。

```text
字典序比较的是从高位到低位的前缀路径；
所以字典序天然适合用 Trie / 前缀树视角理解。
```

## 并查集

并查集的核心直觉：

```text
维护一批元素属于哪个集合。
```

它最擅长两类操作：

```text
find(x)：找到 x 所在集合的代表元
union(a, b)：把 a 和 b 所在集合合并
```

### 怎么想到并查集

可以用这几个问题判断：

- 题目是否在不断合并关系？
- 是否需要频繁判断两个对象是否属于同一组？
- 关系是否具有传递性？例如 `a` 和 `b` 同组，`b` 和 `c` 同组，则 `a` 和 `c` 同组。
- 是否只关心连通性/同组关系，而不关心具体路径？

如果答案大多是 yes，就可以考虑并查集。

常见类型：

| 类型 | 题目信号 | 建模方式 | 代表题 |
| --- | --- | --- | --- |
| 直接连通性 | 加边、成环、连通块数量 | 边表示两个点同组 | 684 |
| 等价关系 | `==`、相似、可替换、同类 | 等价关系合并，冲突关系检查 | 990 |
| 中介点建模 | 同行/同列、共享邮箱、共享因子 | 通过行、列、邮箱、因子把对象连起来 | 947 |
| 带权关系 | 比值、距离、势能差 | 同组之外，还维护到根的关系值 | 399 |

### 普通并查集模板

```python
parent = list(range(n))
size = [1] * n

def find(x):
    if parent[x] != x:
        parent[x] = find(parent[x])
    return parent[x]

def union(a, b):
    ra, rb = find(a), find(b)
    if ra == rb:
        return False
    if size[ra] > size[rb]:
        ra, rb = rb, ra
    parent[ra] = rb
    size[rb] += size[ra]
    return True
```

路径压缩的意义：

```text
把 x 到根路径上的节点直接挂到根下面，让后续 find 更快。
```

### [LCR 118. 冗余连接](https://leetcode.cn/problems/7LpjUW/)

```python
class Solution:
    def findRedundantConnection(self, edges: List[List[int]]) -> List[int]:
        """
        连成连通图后，剩余的那一条边就是可以删去的边。
        """
        n = len(edges)
        father = [i for i in range(n)]

        def find(a):
            if father[a] == a:
                return a
            father[a] = find(father[a])
            return father[a]

        def is_together(a, b):
            return find(a) == find(b)

        def union(a, b):
            u, v = find(a), find(b)
            father[u] = v

        res = []
        for a, b in edges:
            a -= 1
            b -= 1
            if not is_together(a, b):
                union(a, b)
            else:
                res.extend([a + 1, b + 1])
                break
        return res
```

### [399. 除法求值](https://leetcode.cn/problems/evaluate-division/)

这题为什么想到并查集：

```text
a / b = 2.0 表示 a 和 b 有关系；
b / c = 3.0 表示 b 和 c 有关系；
于是 a、b、c 处在同一个连通集合里。
```

普通并查集只能回答：

```text
a 和 c 是否在同一个集合？
```

但本题还要回答：

```text
a / c 等于多少？
```

所以需要带权并查集。

核心定义：

```text
parent[x]：x 的父节点
weight[x]：x / parent[x] 的值
```

路径压缩后，希望保持：

```text
weight[x] = x / root
```

于是如果 `a` 和 `b` 根相同：

```text
a / b = (a / root) / (b / root)
      = weight[a] / weight[b]
```

模板：

```python
class UnionFind:

    def __init__(self):
        self.parent = {}
        self.weight = {}

    def add(self, x):
        if x not in self.parent:
            self.parent[x] = x
            self.weight[x] = 1.0

    def find(self, x):
        if self.parent[x] != x:
            orig_parent = self.parent[x]
            root = self.find(orig_parent)
            self.weight[x] *= self.weight[orig_parent]
            self.parent[x] = root
        return self.parent[x]

    def union(self, a, b, value):
        """
        已知 a / b = value。
        """
        self.add(a)
        self.add(b)

        ra = self.find(a)
        rb = self.find(b)
        if ra == rb:
            return

        self.parent[ra] = rb
        self.weight[ra] = value * self.weight[b] / self.weight[a]

    def connected_ratio(self, a, b):
        if a not in self.parent or b not in self.parent:
            return -1.0

        ra = self.find(a)
        rb = self.find(b)
        if ra != rb:
            return -1.0

        return self.weight[a] / self.weight[b]
```

`union` 公式推导：

```text
已知 a / b = value
已有 weight[a] = a / ra
已有 weight[b] = b / rb
合并时令 parent[ra] = rb，需要求 weight[ra] = ra / rb

a / b
= (a / ra) * (ra / rb) / (b / rb)
= weight[a] * weight[ra] / weight[b]
= value

所以：
weight[ra] = value * weight[b] / weight[a]
```

完整解法：

```python
class Solution:
    def calcEquation(
        self,
        equations: List[List[str]],
        values: List[float],
        queries: List[List[str]],
    ) -> List[float]:
        uf = UnionFind()

        for (a, b), v in zip(equations, values):
            uf.union(a, b, v)

        return [uf.connected_ratio(a, b) for a, b in queries]
```

题后总结：

- 普通并查集维护“是否同组”。
- 带权并查集维护“是否同组 + 节点到根的关系值”。
- `weight[x] = x / parent[x]` 是这题最重要的定义。
- 路径压缩时要同步更新权值，让 `weight[x]` 变成 `x / root`。

## ST 表

ST 表，也叫 Sparse Table，适合处理：

```text
静态数组 + 多次区间查询。
```

它的核心不是“动态维护”，而是“提前把很多区间块算好”。

### 核心直觉

如果要查询 `[l, r]` 的最小值，可以用两个长度相同的 2 的幂区间覆盖：

```text
[l, l + 2^k - 1]
[r - 2^k + 1, r]
```

这两个区间可能重叠，但对最小值、最大值、GCD 这类幂等运算来说，重叠没有影响。

```text
min(x, x) = x
max(x, x) = x
gcd(x, x) = x
```

所以经典 ST 表适合：

- 区间最小值 / 最大值。
- 区间 GCD。
- 其它允许重叠计算的幂等信息。

不适合直接做 `O(1)` 区间和，因为：

```text
sum(x, x) != x
```

### 怎么建表

定义：

```text
st[j][i] = 从 i 开始，长度为 2^j 的区间信息。
```

转移：

```text
长度 2^j 的区间
= 左半段长度 2^(j-1) 的区间
+ 右半段长度 2^(j-1) 的区间
```

代码形状：

```python
n = len(nums)
log = n.bit_length()
st = [[0] * n for _ in range(log)]

for i, x in enumerate(nums):
    st[0][i] = x

for j in range(1, log):
    length = 1 << j
    half = length >> 1
    for i in range(n - length + 1):
        st[j][i] = min(st[j - 1][i], st[j - 1][i + half])
```

### 怎么查询

查询 `[l, r]`：

```python
def query(l: int, r: int) -> int:
    length = r - l + 1
    j = length.bit_length() - 1
    return min(st[j][l], st[j][r - (1 << j) + 1])
```

这里的关键是：

```text
2^j 是不超过区间长度的最大 2 的幂。
```

于是两个长度为 `2^j` 的块能覆盖整个 `[l, r]`。

### 代表题：[402. 移掉 K 位数字](https://leetcode.cn/problems/remove-k-digits/)

这题更常见的做法是单调栈，但也能用 ST 表。

等价转化：

```text
从 n 个数字中保留 n-k 个数字，要求形成的子序列字典序最小。
```

设最终长度为：

```text
m = n - k
```

每次选答案的一位。假设当前还需要选 `need` 位，左边界是 `left`，那么当前位最多只能选到：

```text
right = n - need
```

否则后面就凑不够 `need - 1` 位。

所以每一步都是：

```text
在 [left, right] 中查询最小数字；
如果最小数字有多个，选最靠左的那个。
```

这正好是 ST 表擅长的静态区间最小值查询。为了同时得到最小数字和最靠左下标，表里存 `(字符, 下标)`。

```python
class Solution:
    def removeKdigits(self, num: str, k: int) -> str:
        n = len(num)
        m = n - k
        if m == 0:
            return "0"

        log = n.bit_length()
        st = [[None] * n for _ in range(log)]

        for i, ch in enumerate(num):
            st[0][i] = (ch, i)

        for j in range(1, log):
            length = 1 << j
            half = length >> 1
            for i in range(n - length + 1):
                st[j][i] = min(st[j - 1][i], st[j - 1][i + half])

        def query(l: int, r: int):
            length = r - l + 1
            j = length.bit_length() - 1
            return min(st[j][l], st[j][r - (1 << j) + 1])

        ans = []
        left = 0

        for need in range(m, 0, -1):
            right = n - need
            ch, idx = query(left, right)
            ans.append(ch)
            left = idx + 1

        res = ''.join(ans).lstrip('0')
        return res or "0"
```

和单调栈的区别：

```text
单调栈：扫描过程中反悔最近选择，O(n)。
ST 表：每一位直接在合法区间里查最小值，预处理 O(n log n)，查询总 O(n)。
```

题后总结：

- ST 表适合静态区间查询。
- `st[j][i]` 表示从 `i` 开始、长度为 `2^j` 的区间信息。
- 建表时用两个半区间合并。
- 查询时用两个长度为 `2^j` 的重叠区间覆盖 `[l, r]`。
- 只有幂等信息才能安全重叠覆盖。
