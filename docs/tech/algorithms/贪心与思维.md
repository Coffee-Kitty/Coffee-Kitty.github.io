# 贪心与思维

> 对应灵茶山艾府题单：[贪心与思维](https://leetcode.cn/discuss/post/g6KTKL/)。

## 学习目标

- 能说明每个贪心选择为什么不会影响最优解。
- 能区分排序贪心、区间贪心、反悔贪心、构造和纯思维题。

## 总直觉

DP 和贪心都在处理“选择”。

更准确地说：

```text
DP 可以看作是不重不漏的暴力遍历。
贪心则是在存在某种性质时，对这棵选择树做剪枝。
```

如果没有贪心性质，我们往往需要保留多个状态：

```text
这个选法可能以后更好；
那个选法也不能直接排除；
所以用 DP 把可能性都保存下来。
```

而贪心成立时，说明某个局部选择是“安全的”：

```text
只要这样选，就不会让全局最优变差。
```

于是我们可以剪掉其它选择，只沿着这条局部最优路径走。

所以贪心最重要的不是“选最大”或“选最小”，而是证明：

```text
为什么这个局部选择不会后悔？
```

常见证明方式：

- 交换论证：如果最优解没这么选，也能把它换成我的选择且不变差。
- 领先性：我的方案在每一步都不比任何可行最优方案差。
- 反证法：假设不这么选更优，会推出矛盾。

一句话：

```text
DP 是保留所有必要可能性；
贪心是证明某些可能性永远不需要保留。
```

## 区间贪心

区间贪心常见母问题：

```text
不相交区间：从区间中选尽量多的两两不相交区间。
区间分组：把区间分成最少的组，每组内部互不相交。
区间选点：用最少的点，使每个区间至少包含一个点。
区间覆盖：用尽量少的区间，覆盖指定线段。
```

这类题最常见的安全选择是：

```text
优先保留右端点更小的区间。
```

原因是：

```text
右端点越小，对后续区间的限制越弱。
```

### [435. 无重叠区间](https://leetcode.cn/problems/non-overlapping-intervals/)

题目要求：

```text
删除最少区间，使剩余区间互不重叠。
```

等价于：

```text
保留最多互不重叠区间。
```

核心贪心：

```text
如果两个区间重叠，删除右端点更大的那个；
也就是保留结束时间更早的那个。
```

#### 交换论证

假设有两个重叠区间：

```text
A = [a1, a2]
B = [b1, b2]
```

并且：

```text
a2 <= b2
```

也就是说：

```text
A 结束更早；
B 结束更晚。
```

我们要证明：

```text
保留 A 至少不比保留 B 差。
```

关键观察：

```text
一个区间能不能和后面的区间共存，只取决于它的右端点。
```

如果后面有区间：

```text
C = [c1, c2]
```

并且 `C` 能接在 `B` 后面，那么：

```text
c1 >= b2
```

又因为：

```text
a2 <= b2
```

所以：

```text
c1 >= b2 >= a2
```

因此：

```text
C 也一定能接在 A 后面。
```

也就是说：

```text
所有能接在 B 后面的区间，也都能接在 A 后面。
```

反过来不一定成立。因为 A 结束更早，可能还能接上更多区间。

所以：

```text
保留右端点更小的 A，不会让后续选择变差，只可能更好。
```

换成交换论证：

```text
如果某个最优解保留了 B，没有保留 A；
那么可以把 B 替换成 A。
```

替换后：

```text
区间数量不变；
方案仍然不重叠；
右端点还更小或相同。
```

所以一定存在一个同样好的最优解保留 A。

这证明：

```text
保留右端点更小的区间是安全的。
```

#### 写法一：按右端点排序，保留最多

```python
class Solution:
    def eraseOverlapIntervals(self, intervals: List[List[int]]) -> int:
        intervals.sort(key=lambda x: x[1])

        keep = 0
        end = -10**18

        for l, r in intervals:
            if l >= end:
                keep += 1
                end = r

        return len(intervals) - keep
```

#### 写法二：按左端点排序，冲突时删右端点更大的

```python
class Solution:
    def eraseOverlapIntervals(self, intervals: List[List[int]]) -> int:
        intervals.sort()

        ans = 0
        end = intervals[0][1]

        for l, r in intervals[1:]:
            if l < end:
                ans += 1
                end = min(end, r)
            else:
                end = r

        return ans
```

复盘：

```text
右端点越小，对后续区间的限制越弱；
任何能接在晚结束区间后面的区间，一定也能接在早结束区间后面。
```

#### DP 对照

这题也可以用 DP 做。

转化仍然是：

```text
删除最少区间 = 保留最多互不重叠区间。
```

先按右端点排序。

定义：

```text
dp[i] = 以第 i 个区间作为最后一个保留区间时，最多能保留多少个区间。
```

转移：

```text
dp[i] = 1 + max(dp[j])
```

其中 `j < i`，并且：

```text
intervals[j][1] <= intervals[i][0]
```

也就是第 `j` 个区间能接在第 `i` 个区间前面。

代码：

```python
class Solution:
    def eraseOverlapIntervals(self, intervals: List[List[int]]) -> int:
        intervals.sort(key=lambda x: x[1])
        n = len(intervals)

        dp = [1] * n

        for i in range(n):
            for j in range(i):
                if intervals[j][1] <= intervals[i][0]:
                    dp[i] = max(dp[i], dp[j] + 1)

        keep = max(dp)
        return n - keep
```

复杂度：

```text
O(n^2)
```

这个 DP 的含义是：

```text
对每个区间 i，都尝试所有可能接在它前面的区间 j。
```

而贪心能做，是因为我们证明了：

```text
右端点更小的选择一定不比右端点更大的选择差。
```

所以：

```text
DP：枚举所有可能的前驱。
贪心：证明某类前驱足够安全，于是剪掉其它可能。
```

### [2406. 将区间分为最少组数](https://leetcode.cn/problems/divide-intervals-into-minimum-number-of-groups/)

题目要求：

```text
把所有区间分成最少的组，使每组内部区间两两不相交。
```

核心转化：

```text
最少组数 = 最大同时重叠区间数。
```

为什么：

```text
如果某个时间点同时被 k 个区间覆盖，这 k 个区间两两冲突，必须放进 k 个不同组。
所以答案至少是最大重叠数。

反过来，按左端点扫描区间时，能接到已有组就接，不能接才新开组。
需要的新组数正好由同时占用的最大组数决定。
```

#### 写法一：最小堆模拟分组

```python
from heapq import heappush, heapreplace

class Solution:
    def minGroups(self, intervals: List[List[int]]) -> int:
        intervals.sort()
        heap = []

        for left, right in intervals:
            if heap and left > heap[0]:
                heapreplace(heap, right)
            else:
                heappush(heap, right)

        return len(heap)
```

堆的含义：

```text
heap 中存每个组当前最后一个区间的 right。
heap[0] 是结束最早的组。
```

判断条件：

```text
left > heap[0]
```

因为本题是闭区间，`[1,5]` 和 `[5,8]` 在 `5` 这个点相交，不能放在同一组。

直觉：

```text
如果结束最早的组都接不了当前区间，其它结束更晚的组也接不了；
所以只能新开一组。
```

#### 写法二：差分 / 上下车模型

把每个区间看成一个人：

```text
left 时刻上车；
right + 1 时刻下车。
```

同时在车上的最大人数，就是最大重叠区间数。

```python
class Solution:
    def minGroups(self, intervals: List[List[int]]) -> int:
        events = []

        for left, right in intervals:
            events.append((left, 1))
            events.append((right + 1, -1))

        events.sort()

        cur = ans = 0
        for _, delta in events:
            cur += delta
            ans = max(ans, cur)

        return ans
```

这里用 `right + 1`，是为了处理闭区间。

如果不用 `right + 1`，而是把下车事件放在 `right`，那么同一时刻需要保证先下车再上车：

```python
events.sort(key=lambda x: (x[0], x[1]))
```

由于 `-1 < 1`，同一时间点会先处理下车，再处理上车。

复盘：

```text
区间分组 = 最大同时重叠数。
需要具体构造分组时，用小根堆维护每个组的结束时间。
只问最少组数时，用差分 / 扫描线统计最大重叠数更直接。
```

### [452. 用最少数量的箭引爆气球](https://leetcode.cn/problems/minimum-number-of-arrows-to-burst-balloons/)

题目要求：

```text
每个气球是一个闭区间 [left, right]。
一支箭射在坐标 x，可以引爆所有满足 left <= x <= right 的气球。
求最少需要多少支箭。
```

母问题：

```text
给一堆区间，放尽量少的点，使每个区间至少包含一个点。
```

核心贪心：

```text
按右端点从小到大排序。
每次遇到一个还没被当前点覆盖的区间，就在它的右端点放一个点。
```

为什么放右端点：

```text
当前最早结束的区间必须被某个点打中；
点必须放在它的区间内部。
在不丢掉当前区间的前提下，把点放得越靠右，越可能顺便覆盖后面的区间。
```

代码：

```python
class Solution:
    def findMinArrowShots(self, points: List[List[int]]) -> int:
        points.sort(key=lambda x: x[1])

        ans = 0
        arrow = -10**18

        for left, right in points:
            if left > arrow:
                ans += 1
                arrow = right

        return ans
```

和 435 的区别：

```text
435 维护的是已选最后一个区间的右端点 end。
如果新区间不重叠，就选择新区间，并更新 end。

452 维护的是一个已经确定的点 arrow。
如果新区间包含 arrow，说明它已经被覆盖，不能把 arrow 随便更新为当前区间右端点。
```

为什么不能更新 `arrow`：

```text
arrow 是已经射出去的点。
如果当前区间已经包含 arrow，它已经被这支箭打中。
此时把 arrow 改成当前区间的 right，可能会失去对前面区间的覆盖。
```

例如：

```text
[1,6], [2,8]
```

第一支箭放在 `6`。第二个区间包含 `6`，所以已经被打中。
如果把箭改成 `8`，它就不再打中 `[1,6]`。

复盘：

```text
区间选点 = 按右端点排序，在最早结束区间的右端点放点。
当前点已经确定；后续区间能被它覆盖，就不要更新这个点。
```

### [1326. 灌溉花园的最少水龙头数目](https://leetcode.cn/problems/minimum-number-of-taps-to-open-to-water-a-garden/)

题目要求：

```text
给定一条线段 [0, n]，每个水龙头都能覆盖一个区间。
问最少打开多少个水龙头，才能覆盖整条线段。
```

母问题：

```text
给定一堆区间，从中选出尽量少的区间，覆盖目标线段。
```

核心贪心：

```text
从左到右扫描当前已经覆盖到的位置。
每次在所有左端点不超过当前覆盖边界的区间里，选择右端点最远的那个。
```

为什么选最远右端点：

```text
当前必须先把左边这一段补上；
在所有能接上的区间里，右端点越远，后面剩下的未覆盖部分越少。
```

这题常转成跳跃游戏的形式：

```text
把每个区间看成一个“从左端点出发，能跳到最远右端点”的跳板。
```

常见做法是先把每个位置能到达的最远右端点预处理出来，再做一次线性扫描：

```python
class Solution:
    def minTaps(self, n: int, ranges: List[int]) -> int:
        far = [0] * (n + 1)
        for i, r in enumerate(ranges):
            l = max(0, i - r)
            rr = min(n, i + r)
            far[l] = max(far[l], rr)

        ans = 0
        cur_end = 0
        next_end = 0

        for i in range(n):
            next_end = max(next_end, far[i])
            if i == cur_end:
                if next_end <= i:
                    return -1
                ans += 1
                cur_end = next_end

        return ans
```

直觉：

```text
cur_end 表示当前已经覆盖到哪里；
next_end 表示在当前覆盖范围内，下一步能扩到多远。
当走到 cur_end 时，必须做一次选择，把覆盖边界推进到 next_end。
```

复盘：

```text
区间覆盖 = 从左往右贪心扩展当前覆盖边界；
每次选能把右边界推得最远的区间。
```

## 反悔贪心

反悔贪心和延迟决策贪心有一个共同模型：

```text
堆维护历史候选。
当约束不满足、或者需要做选择时，从历史候选中取出当前最优项。
```

反悔贪心的常见模型：

```text
先尽量选；
如果后来发现违反约束，就从已经选过的对象里删掉最不划算的那个。
```

它通常会配合堆：

```text
堆维护当前已选集合中最应该被反悔的元素。
```

### [630. 课程表 III](https://leetcode.cn/problems/course-schedule-iii/)

题目：

```text
courses[i] = [duration, lastDay]
duration：课程耗时
lastDay：最晚完成时间
```

目标：

```text
最多能上多少门课。
```

关键卡点：

```text
已选课程的具体顺序不是固定的，可以重新编排。
```

所以当按 `lastDay` 从小到大处理课程时，真正要维护的是：

```text
当前已选课程的总耗时 total。
```

如果：

```text
total <= 当前 lastDay
```

说明当前已选课程可以安排进当前截止时间限制内。

贪心过程：

```text
1. 按 lastDay 从小到大排序。
2. 先选择当前课程，把 duration 加入 total。
3. 用最大堆维护已选课程的 duration。
4. 如果 total > 当前 lastDay，说明必须反悔删掉一门课。
5. 删除已选课程中 duration 最大的那门。
```

为什么删最长课程是安全的：

```text
超时后必须删一门课。
无论删哪门，课程数量都少 1。
既然数量一样，就应该让剩余总耗时尽量小。
删掉 duration 最大的课程，能让 total 下降最多。
```

交换论证：

假设某个最优方案超时后删掉的不是最长课程，而是某门较短课程 `x`。

设最长课程是 `L`，并且：

```text
duration[L] >= duration[x]
```

原方案：

```text
删 x，保留 L。
```

交换成：

```text
删 L，保留 x。
```

课程数量不变。总耗时变为：

```text
原总耗时 - duration[L] + duration[x]
```

由于：

```text
duration[L] >= duration[x]
```

所以新总耗时不更大。

因此：

```text
如果删 x 后可行，那么删 L 后也一定可行；
并且对后续更晚的 lastDay 也不更差。
```

这证明了：

```text
当必须反悔删一门课时，删 duration 最大的课程是安全的。
```

代码：

```python
class Solution:
    def scheduleCourse(self, courses: List[List[int]]) -> int:
        courses.sort(key=lambda x: x[1])

        total = 0
        heap = []

        for duration, last_day in courses:
            total += duration
            heappush(heap, -duration)

            if total > last_day:
                total += heappop(heap)

        return len(heap)
```

注意：

```text
Python 的 heapq 是小根堆；
这里放 -duration，相当于维护最大堆。
total += heappop(heap) 等价于 total -= 最大 duration。
```

复盘：

```text
630 = 按截止时间排序 + 先选 + 超时后反悔删最长课程。
```

最重要的理解：

```text
已选课程可以自由编排，所以只需比较 total 和当前 lastDay。
```

### [871. 最低加油次数](https://leetcode.cn/problems/minimum-number-of-refueling-stops/)

这题也使用堆维护历史候选，但更准确地说，它是：

```text
延迟决策贪心。
```

它和 630 的区别：

```text
630：先选课程，超时后撤销最差的。
871：路过加油站时先不选，走不动时启用最好的。
```

关键思考方式：

```text
能晚做决定，就晚做决定。
```

路过一个加油站时，不必立刻决定要不要加油：

```text
先把这个站的油量放进候选池；
等真的油不够了，再从所有已经路过的候选里选油最多的。
```

这比“每到一个站就决定加或不加”更好，因为后者会产生大量分支。

母问题：

```text
不断向前推进；
遇到候选先存起来；
当资源不够时，从历史候选中取出最优项补救。
```

代码：

```python
class Solution:
    def minRefuelStops(
        self,
        target: int,
        startFuel: int,
        stations: List[List[int]],
    ) -> int:
        stations.append([target, 0])

        fuel = startFuel
        prev = 0
        heap = []
        ans = 0

        for pos, gas in stations:
            fuel -= pos - prev

            while fuel < 0:
                if not heap:
                    return -1
                fuel += -heappop(heap)
                ans += 1

            heappush(heap, -gas)
            prev = pos

        return ans
```

代码含义：

```text
每到一个位置，先消耗从 prev 到 pos 的油。
如果油不够，就从之前路过的站里取最大油量补。
当前站能到达后，再把当前站油量加入候选池。
```

为什么取最大油量：

```text
每次加油次数都 +1；
既然次数一样，就应该让这一次补给带来最大的前进能力。
```

复盘：

```text
871 = 延迟决策 + 历史候选池 + 缺资源时取最大补救。
```

最重要的理解：

```text
不是每遇到一个站就判断加不加；
而是把路过的站放进堆，等必须加油时再选最大。
```

### [502. IPO](https://leetcode.cn/problems/ipo/)

这题也是堆维护动态候选。

题意：

```text
初始资金是 w；
每个项目有启动门槛 capital[i] 和收益 profits[i]；
当前资金 w >= capital[i] 时，才能做这个项目；
最多做 k 个项目；
求最终最大资金。
```

核心贪心：

```text
当前能做的项目里，选择 profit 最大的。
```

关键点：

```text
每做完一个项目，资金会增加；
资金增加后，可能解锁更多项目；
所以候选集合需要持续更新。
```

流程：

```text
1. 按 capital 从小到大排序项目。
2. 用指针 i 扫描项目。
3. 每一轮，把 capital <= 当前资金 w 的项目全部加入最大堆。
4. 从最大堆中取 profit 最大的项目做。
5. 最多重复 k 次。
```

为什么选最大 profit 是安全的：

```text
当前堆里的项目都已经可做；
做项目不会消耗资金，只会增加资金；
选择 profit 最大的项目，会让当前资金增长最多；
不会让任何已可做项目变得不可做，只可能解锁更多项目。
```

代码：

```python
class Solution:
    def findMaximizedCapital(
        self,
        k: int,
        w: int,
        profits: List[int],
        capital: List[int],
    ) -> int:
        projects = sorted(zip(capital, profits))
        heap = []
        i = 0
        n = len(projects)

        for _ in range(k):
            while i < n and projects[i][0] <= w:
                heappush(heap, -projects[i][1])
                i += 1

            if not heap:
                break

            w += -heappop(heap)

        return w
```

和 871 的关系：

```text
871：资源不够时，从历史候选里取最大补救。
502：每轮主动从当前可行候选里取最大收益。
```

统一模型：

```text
候选池动态变化；
堆维护当前最优候选。
```

### [135. 分发糖果](https://leetcode.cn/problems/candy/)

这题的核心不是单纯“当前和前一个比一下”，而是：

```text
双向相邻约束，需要拆成两个单向约束再合并。
```

题目约束：

```text
每个孩子至少 1 颗糖；
评分更高的孩子，必须比相邻评分低的孩子糖更多。
```

相邻约束有两个方向：

```text
如果 ratings[i] > ratings[i - 1]，那么 candies[i] > candies[i - 1]。
如果 ratings[i] > ratings[i + 1]，那么 candies[i] > candies[i + 1]。
```

所以分两遍：

```text
左到右：处理上升坡。
右到左：处理下降坡。
```

代码：

```python
class Solution:
    def candy(self, ratings: List[int]) -> int:
        n = len(ratings)
        candies = [1] * n

        for i in range(1, n):
            if ratings[i] > ratings[i - 1]:
                candies[i] = candies[i - 1] + 1

        for i in range(n - 2, -1, -1):
            if ratings[i] > ratings[i + 1]:
                candies[i] = max(candies[i], candies[i + 1] + 1)

        return sum(candies)
```

为什么第二遍要用 `max`：

```text
一个位置可能同时受左边和右边约束；
第二遍不能破坏第一遍已经满足的上升坡约束。
```

反例直觉：

```text
ratings = [5, 4, 3, 2, 1]
```

下降链条的信息必须从右往左传播。只从左往右处理，会出现连锁失效。

和栈的关系：

```text
它和栈都在处理连续局部关系的传播和结算；
但这题约束只来自左右相邻，用两遍扫描比显式用栈更自然。
```

复盘：

```text
135 = 双向局部约束 -> 两个单向扫描 -> max 合并。
```

### [406. 根据身高重建队列](https://leetcode.cn/problems/queue-reconstruction-by-height/)

这题是排序贪心 / 插入构造。

题意：

```text
people[i] = [h, k]
h 表示身高；
k 表示前面身高 >= h 的人数。
```

关键观察：

```text
一个人的 k 只关心前面有多少个身高 >= 自己的人。
高个子不会被矮个子影响。
```

所以先处理高个子：

```text
身高 h 从高到低排序；
同身高时，k 从小到大排序。
```

然后按顺序插入：

```text
把每个人插入到下标 k 的位置。
```

代码：

```python
class Solution:
    def reconstructQueue(self, people: List[List[int]]) -> List[List[int]]:
        people.sort(key=lambda x: (-x[0], x[1]))

        ans = []
        for p in people:
            ans.insert(p[1], p)

        return ans
```

为什么插入下标 `k` 是对的：

```text
当处理 [h, k] 时，队列里已经放进去的人身高都 >= h；
所以把他插到下标 k，就保证前面刚好有 k 个身高 >= h 的人。
```

为什么后续不会破坏它：

```text
后面插入的人都更矮；
更矮的人不会计入“身高 >= h 的人数”；
所以不会影响已经插入的人的 k。
```

复盘：

```text
406 = 先处理不会被后续影响的人。
```

更具体地说：

```text
先高后矮；
高个子的约束先确定；
矮个子后插入不会影响高个子的 k。
```
