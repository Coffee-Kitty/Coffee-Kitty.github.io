## 核心直觉

位运算不要只背技巧。更稳定的理解是：

```text
一个整数可以看成若干个二进制位组成的状态。
```

很多位运算题的突破口是：

```text
1. 每一位能不能独立判断？
2. 有没有恒等式可以让元素抵消？
3. 能不能从高位到低位构造答案？
4. 能不能用一个整数表示一个集合？
```

## 异或 XOR

异或是位运算里最常见的模型之一。

必须记住的性质：

```text
x ^ x = 0
x ^ 0 = x
x ^ y = y ^ x
(x ^ y) ^ z = x ^ (y ^ z)
```

所以异或适合处理：

```text
成对抵消；
奇偶性；
前缀异或；
找出不同元素。
```

### [136. 只出现一次的数字](https://leetcode.cn/problems/single-number/)

题型信号：

```text
每个元素都出现两次，只有一个元素出现一次。
```

直接全部异或：

```text
相同元素会抵消成 0；
剩下的就是只出现一次的元素。
```

例子：

```text
nums = [4, 1, 2, 1, 2]

4 ^ 1 ^ 2 ^ 1 ^ 2
= 4 ^ (1 ^ 1) ^ (2 ^ 2)
= 4
```

代码：

```python
class Solution:
    def singleNumber(self, nums: List[int]) -> int:
        ans = 0
        for x in nums:
            ans ^= x
        return ans
```

复盘：

```text
136 = 异或的成对抵消模型。
```

### [260. 只出现一次的数字 III](https://leetcode.cn/problems/single-number-iii/)

题型信号：

```text
有两个元素只出现一次，其余元素都出现两次。
```

假设两个只出现一次的元素是 `a` 和 `b`。

第一步，全部异或：

```text
xor_all = a ^ b
```

这一步不能直接得到答案，因为答案有两个数。但由于 `a != b`，所以 `a ^ b` 至少有一位是 `1`。

```text
xor_all 的某一位是 1
=> a 和 b 在这一位上不同
=> 可以用这一位把 a 和 b 分到不同组
```

第二步，用 `lowbit` 找到 `xor_all` 中最低位的 `1`：

```python
lowbit = xor_all & -xor_all
```

这句的作用是：

```text
取出 x 二进制中最低位的 1。
```

例子：

```text
x = 6
二进制：110
最低位的 1 是：010

6 & -6 = 2
```

为什么它能做到：

```text
-x = ~x + 1
```

以 8 位表示为例：

```text
x  = 00000110
~x = 11111001
-x = 11111010

  00000110
& 11111010
= 00000010
```

第三步，按 `lowbit` 分组。

```text
x & lowbit != 0 的一组；
x & lowbit == 0 的一组。
```

相同数字一定会进入同一组，因为相同数字的每一位都一样。于是每组内部继续成对抵消。

例子：

```text
nums = [1, 2, 1, 3, 2, 5]

两个特殊数是 3 和 5：
3 = 011
5 = 101
3 ^ 5 = 110

lowbit = 010
```

按这一位分组：

```text
这一位是 0：1, 1, 5
这一位是 1：2, 3, 2
```

组内异或：

```text
1 ^ 1 ^ 5 = 5
2 ^ 3 ^ 2 = 3
```

代码：

```python
class Solution:
    def singleNumber(self, nums: List[int]) -> List[int]:
        xor_all = 0
        for x in nums:
            xor_all ^= x

        lowbit = xor_all & -xor_all

        a = 0
        b = 0
        for x in nums:
            if x & lowbit:
                a ^= x
            else:
                b ^= x

        return [a, b]
```

复杂度：

```text
时间复杂度：O(n)
空间复杂度：O(1)
```

复盘：

```text
136：全部异或，成对抵消，剩一个。
260：全部异或得到 a ^ b，再用差异位分组，每组内部继续抵消。
```

`lowbit` 在这题里不是答案，而是分组工具。

## 最低位的 1

和 `lowbit` 相关的一组技巧：

```text
x & -x       取出 x 最低位的 1
x & (x - 1) 删除 x 最低位的 1
```

### [191. 位1的个数](https://leetcode.cn/problems/number-of-1-bits/)

题型信号：

```text
统计一个整数二进制中有多少个 1。
```

最直观做法是右移：

```python
class Solution:
    def hammingWeight(self, n: int) -> int:
        ans = 0
        while n:
            ans += n & 1
            n >>= 1
        return ans
```

这个写法的含义是：

```text
每次检查最低位是不是 1；
然后把 n 右移一位。
```

另一种写法是每次删除最低位的 `1`：

```python
class Solution:
    def hammingWeight(self, n: int) -> int:
        ans = 0
        while n:
            n &= n - 1
            ans += 1
        return ans
```

这句：

```python
n &= n - 1
```

等价于：

```text
删除 n 最低位的 1，并保持其它更高位不变。
```

也可以先显式取出 `lowbit`，再用减法删掉它：

```python
while n:
    lowbit = n & -n
    n -= lowbit
    ans += 1
```

例子：

```text
n = 110100

lowbit = 000100
n -= lowbit
=> 110000

lowbit = 010000
n -= lowbit
=> 100000

lowbit = 100000
n -= lowbit
=> 000000
```

所以这题有两种直觉：

```text
右移：逐位检查。
lowbit / n&(n-1)：每次删掉一个已经确认存在的 1。
```

当前理解：

```text
如果只是统计 1 的个数，右移更直观；
如果要练位运算技巧，记住 x & (x - 1) 可以删除最低位的 1。
```

### [231. 2 的幂](https://leetcode.cn/problems/power-of-two/)

题型信号：

```text
判断一个数是否只有一个二进制位是 1。
```

如果 `n` 是 2 的幂，那么二进制中只有一个 `1`：

```text
1  = 0001
2  = 0010
4  = 0100
8  = 1000
16 = 10000
```

此时：

```text
n - 1 会把唯一的 1 变成 0，并把它右边所有位变成 1。
```

例子：

```text
8     = 1000
8 - 1 = 0111

1000 & 0111 = 0000
```

所以：

```text
如果 n 是 2 的幂，n & (n - 1) == 0。
```

如果 `n` 不是 2 的幂，例如：

```text
12     = 1100
12 - 1 = 1011

1100 & 1011 = 1000
```

这只是删掉了最低位的 `1`，高位仍然还有 `1`，所以结果不是 `0`。

代码：

```python
class Solution:
    def isPowerOfTwo(self, n: int) -> bool:
        return n > 0 and (n & (n - 1)) == 0
```

注意必须判断 `n > 0`：

```text
0 不是 2 的幂；
但 0 & (0 - 1) 也会得到 0。
```

复盘：

```text
2 的幂 = 二进制中只有一个 1。
x & (x - 1) 会删除最低位的 1。
如果删完后变成 0，说明原来只有一个 1。
```

### [338. 比特位计数](https://leetcode.cn/problems/counting-bits/)

这题是 [191. 位1的个数](https://leetcode.cn/problems/number-of-1-bits/) 的批量版。

题意：

```text
给定 n，返回 0 到 n 每个整数的二进制 1 的个数。
```

注意：返回的是数组，不是求和。

```text
ans[i] = i 的二进制中 1 的个数
```

核心递推来自：

```text
i & (i - 1) 会删除 i 最低位的 1。
```

所以：

```text
i 的 1 的个数
= 删除最低位 1 后的 1 的个数 + 1
```

也就是：

```python
ans[i] = ans[i & (i - 1)] + 1
```

例子：

```text
i = 12
12 = 1100
12 & 11 = 1000

ans[12] = ans[8] + 1
```

递归边界：

```text
ans[0] = 0
```

递推方向：

```text
从小到大枚举 i = 1..n。
```

为什么可以从小到大：

```text
i & (i - 1) 一定小于 i。
```

所以当计算 `ans[i]` 时，`ans[i & (i - 1)]` 一定已经算过。

迭代写法：

```python
class Solution:
    def countBits(self, n: int) -> List[int]:
        ans = [0] * (n + 1)

        for i in range(1, n + 1):
            ans[i] = ans[i & (i - 1)] + 1

        return ans
```

也可以用递归 + 缓存理解：

```python
class Solution:
    def countBits(self, n: int) -> List[int]:
        @cache
        def f(x: int) -> int:
            if x == 0:
                return 0
            return f(x & (x - 1)) + 1

        return [f(i) for i in range(n + 1)]
```

不过本题更推荐迭代写法，因为依赖方向很清楚：

```text
i 依赖一个比 i 小的状态。
```

复盘：

```text
这是一种位运算 DP。
状态不是来自 i-1，而是来自删除最低位 1 后的数。
```

## 拆位统计

拆位统计的核心是：

```text
如果每一位互不影响，就可以逐位统计，再把答案拼回去。
```

它和前面的异或抵消不一样。异或抵消是在“整体上”用恒等式消掉元素；拆位统计是把每个数拆成二进制位，分别看每一位出现了多少次。

### [137. 只出现一次的数字 II](https://leetcode.cn/problems/single-number-ii/)

题型信号：

```text
除了一个元素只出现一次，其余元素都出现三次。
```

这题不能继续用普通异或，因为：

```text
x ^ x = 0
但 x ^ x ^ x = x
```

所以出现三次的数字不会被普通异或抵消掉。

正确思路是拆位统计：

```text
逐位统计所有数字在这一位上有多少个 1。
出现三次的数字，在每一位上的贡献都是 3 的倍数。
只出现一次的数字，会让某些位多出 1。
```

因此：

```text
cnt[bit] % 3
```

就是答案在这一位上是 `0` 还是 `1`。

例子：

```text
nums = [2, 2, 3, 2]

2 = 10
2 = 10
3 = 11
2 = 10
```

第 0 位：

```text
0 + 0 + 1 + 0 = 1
1 % 3 = 1
答案第 0 位是 1
```

第 1 位：

```text
1 + 1 + 1 + 1 = 4
4 % 3 = 1
答案第 1 位是 1
```

所以答案是：

```text
11 = 3
```

代码：

```python
class Solution:
    def singleNumber(self, nums: List[int]) -> int:
        ans = 0

        for bit in range(32):
            cnt = 0
            for x in nums:
                cnt += (x >> bit) & 1

            if cnt % 3:
                ans |= 1 << bit

        if ans >= 1 << 31:
            ans -= 1 << 32

        return ans
```

为什么要处理负数：

```text
题目中的数字可能是负数；
Python 的整数不是固定 32 位；
所以如果第 31 位是 1，需要把它还原成负数。
```

这句：

```python
if ans >= 1 << 31:
    ans -= 1 << 32
```

是在做 32 位有符号整数还原。

复盘：

```text
137 = 拆位统计 + 每位计数 mod 3。
```

关键变化：

```text
136/260：靠异或恒等式整体抵消。
137：普通异或不能抵消三次，所以改成逐位计数。
```

### [477. 汉明距离总和](https://leetcode.cn/problems/total-hamming-distance/)

笨方法：

```text
枚举每一对数；
计算 nums[i] ^ nums[j]；
统计异或结果里有多少个 1。
```

代码形状：

```python
ans = 0

for i in range(n):
    for j in range(i + 1, n):
        x = nums[i] ^ nums[j]
        while x:
            lowbit = x & -x
            x -= lowbit
            ans += 1
```

这里：

```text
x & -x 等价于 x & (~x + 1)，用于取出最低位的 1。
x -= lowbit 用于删除这个最低位的 1。
```

也可以简写成：

```python
while x:
    x &= x - 1
    ans += 1
```

但这个做法复杂度太高：

```text
枚举数对 O(n^2)，每对再统计 bit。
```

优化思路是拆位统计贡献。

对某一位 `bit`：

```text
如果有 c 个数这一位是 1；
有 n - c 个数这一位是 0；
```

那么只有 `1` 和 `0` 配对时，汉明距离在这一位才贡献 `1`。

所以这一位的贡献是：

```text
c * (n - c)
```

把所有位的贡献加起来即可。

例子：

```text
nums = [4, 14, 2]

4  = 0100
14 = 1110
2  = 0010
```

逐位看：

```text
第 0 位：0,0,0 -> c = 0，贡献 0 * 3 = 0
第 1 位：0,1,1 -> c = 2，贡献 2 * 1 = 2
第 2 位：1,1,0 -> c = 2，贡献 2 * 1 = 2
第 3 位：0,1,0 -> c = 1，贡献 1 * 2 = 2
```

总贡献：

```text
0 + 2 + 2 + 2 = 6
```

代码：

```python
class Solution:
    def totalHammingDistance(self, nums: List[int]) -> int:
        n = len(nums)
        ans = 0

        for bit in range(30):
            cnt = 0
            for x in nums:
                cnt += (x >> bit) & 1

            ans += cnt * (n - cnt)

        return ans
```

复盘：

```text
477 = 拆位统计 + 每一位单独计算贡献。
```

关键转化：

```text
不要枚举每一对数；
改成枚举每一位，看这一位能贡献多少对不同。
```

## 前缀异或

前缀异或和前缀和的思想类似，只是运算从加法变成异或。

定义：

```text
pre[0] = 0
pre[i + 1] = pre[i] ^ arr[i]
```

那么：

```text
arr[l..r] 的异或 = pre[r + 1] ^ pre[l]
```

原因是相同前缀会异或抵消。

### [1310. 子数组异或查询](https://leetcode.cn/problems/xor-queries-of-a-subarray/)

这题是前缀异或模板题。

```python
class Solution:
    def xorQueries(self, arr: List[int], queries: List[List[int]]) -> List[int]:
        n = len(arr)
        pre = [0] * (n + 1)

        for i, x in enumerate(arr):
            pre[i + 1] = pre[i] ^ x

        return [pre[r + 1] ^ pre[l] for l, r in queries]
```

复盘：

```text
前缀和：区间和 = pre[r+1] - pre[l]。
前缀异或：区间异或 = pre[r+1] ^ pre[l]。
```

### [1442. 形成两个异或相等数组的三元组数目](https://leetcode.cn/problems/count-triplets-that-can-form-two-arrays-of-equal-xor/)

题目定义：

```text
a = arr[i] ^ ... ^ arr[j-1]
b = arr[j] ^ ... ^ arr[k]
```

要求：

```text
a == b
```

关键证明：

```text
arr[i..k] 的整体异或
= (arr[i] ^ ... ^ arr[j-1]) ^ (arr[j] ^ ... ^ arr[k])
= a ^ b
```

如果：

```text
arr[i..k] 的整体异或 = 0
```

那么：

```text
a ^ b = 0
=> a == b
```

所以只要大区间 `arr[i..k]` 异或为 `0`，`j` 在 `i+1..k` 的任意合法切分都成立。

固定 `i, k` 时，合法切分点：

```text
j = i+1, i+2, ..., k
```

数量是：

```text
k - i
```

用前缀异或：

```text
arr[i..k] 异或 = pre[k + 1] ^ pre[i]
```

所以：

```text
arr[i..k] 异或为 0
<=> pre[k + 1] == pre[i]
```

清晰版本代码：

```python
class Solution:
    def countTriplets(self, arr: List[int]) -> int:
        n = len(arr)
        pre = [0] * (n + 1)

        for i, x in enumerate(arr):
            pre[i + 1] = pre[i] ^ x

        ans = 0

        for i in range(n):
            for k in range(i + 1, n):
                if pre[i] == pre[k + 1]:
                    ans += k - i

        return ans
```

这题 `n <= 300`，所以 `O(n^2)` 可以通过。这个版本最适合理解关系。

哈希优化思路：

```text
如果当前前缀下标是 t，也就是 pre[t]；
那么 k = t - 1；
需要统计所有旧位置 i，满足 pre[i] == pre[t]；
每个 i 的贡献是 k - i = t - 1 - i。
```

所以可以维护：

```text
cnt[x]：前缀异或值 x 出现过多少次
total[x]：这些前缀下标 i 的总和
```

批量贡献：

```text
(t - 1) * cnt[x] - total[x]
```

但本题优先记清晰版本：

```text
pre[i] == pre[k+1] 时，贡献 k-i。
```

## 高位试填

高位试填的核心是：

```text
二进制数值由高位优先决定。
如果要最大化答案，就从高位到低位，尽量让当前位变成 1。
```

这和字典序有点像：

```text
字典序：越靠前的字符越重要。
二进制：越高的位越重要。
```

### [421. 数组中两个数的最大异或值](https://leetcode.cn/problems/maximum-xor-of-two-numbers-in-an-array/)

这题有两种常见理解：

- 异或 Trie。
- 高位试填 + 哈希集合。

我们先记录异或 Trie 直觉。

异或最大化时，每一位都希望两个数不同：

```text
0 ^ 1 = 1
1 ^ 0 = 1
```

所以对每个数 `x`：

```text
Trie 里存已经出现过的历史数字；
查询 x 时，从高位到低位走；
每一位优先找和 x 当前位相反的分支；
如果有相反分支，这一位异或结果就是 1；
如果没有，只能走相同分支，这一位异或结果就是 0。
```

流程：

```text
1. 先把历史数字放进二进制 Trie。
2. 当前数字 x 来了以后，在 Trie 中找一个历史数字 y，使 x ^ y 最大。
3. 查询时从高位到低位，能走相反位就走相反位。
4. 得到当前 x 的最大异或值，更新全局答案。
5. 再把 x 插入 Trie，供后面的数查询。
```

关键直觉：

```text
不是只看某一位，而是从最高位到最低位逐位贪心。
高位能变成 1，就一定优先让高位变成 1。
```

复盘：

```text
421 = 高位优先 + 相反位贪心 + 历史数字 Trie。
```

之后如果要写代码，再回到 [常用数据结构.md](常用数据结构.md) 里的异或 Trie 模板复习。

## 位集合 / 状态压缩

位集合的核心是：

```text
一个整数 mask 可以表示一个集合。
mask 的第 i 位是 1，表示选了第 i 个元素；
mask 的第 i 位是 0，表示没选第 i 个元素。
```

这种表示法适合元素个数不多、但要枚举“选或不选”状态的问题。

### [78. 子集](https://leetcode.cn/problems/subsets/)

这题有三种常见写法。本质复杂度差不多，因为答案本身就有 `2^n` 个子集，把所有子集真正生成出来需要：

```text
O(n * 2^n)
```

区别主要是思考视角和代码形态。

#### 方法一：选或不选

站在输入 `nums[i]` 的角度：

```text
这个元素选，还是不选？
```

代码形状：

```python
class Solution:
    def subsets(self, nums: List[int]) -> List[List[int]]:
        n = len(nums)
        ans = []
        path = []

        def dfs(i: int) -> None:
            if i == n:
                ans.append(path.copy())
                return

            dfs(i + 1)

            path.append(nums[i])
            dfs(i + 1)
            path.pop()

        dfs(0)
        return ans
```

这个模型像一棵二叉决策树：

```text
不选 nums[i]
选 nums[i]
```

#### 方法二：枚举选哪个

站在答案 `path` 的角度：

```text
下一个要加入 path 的元素是谁？
```

代码形状：

```python
class Solution:
    def subsets(self, nums: List[int]) -> List[List[int]]:
        n = len(nums)
        ans = []
        path = []

        def dfs(i: int) -> None:
            ans.append(path.copy())

            for j in range(i, n):
                path.append(nums[j])
                dfs(j + 1)
                path.pop()

        dfs(0)
        return ans
```

这里“不选某些元素”隐含在 `j` 的跳跃中：

```text
如果直接选 nums[j]，就相当于跳过了 nums[i..j-1]。
```

#### 方法三：二进制枚举

站在集合状态 `mask` 的角度：

```text
mask 的第 i 位是 1，表示选 nums[i]。
```

代码形状：

```python
class Solution:
    def subsets(self, nums: List[int]) -> List[List[int]]:
        ans = []
        n = len(nums)

        for mask in range(1 << n):
            subset = []
            for i in range(n):
                if (mask >> i) & 1:
                    subset.append(nums[i])
            ans.append(subset)

        return ans
```

三种视角：

```text
选或不选：输入视角，我正在处理 nums[i]。
枚举选哪个：答案视角，我正在决定 path 的下一个元素。
二进制枚举：状态视角，我正在枚举一个集合 mask。
```

复盘：

```text
78 本身不是难题，但它建立了“整数表示集合”的模型。
这个模型是状态压缩 DP 的基础。
```

### [1125. 最小的必要团队](https://leetcode.cn/problems/smallest-sufficient-team/)

这题是状态压缩 DP 的典型题。

核心建模：

```text
每个技能是一个 bit。
一个人的技能集合是一个 mask。
团队能力 = 所有成员技能 mask 的按位或 OR。
目标是覆盖 full = (1 << m) - 1。
```

比如技能是：

```text
["java", "python", "go"]
```

映射成：

```text
java   -> 第 0 位
python -> 第 1 位
go     -> 第 2 位
```

某个人会：

```text
["java", "go"]
```

那么他的技能 mask 是：

```text
101
```

团队加入这个人，就是：

```text
new_mask = mask | person_mask
```

这题可以理解成“每个人选或不选”，但不要直接枚举所有人的子集。更好的状态是：

```text
当前已经覆盖了哪些技能。
```

用 `@cache` 写法时，可以让递归函数表示：

```text
dfs(mask) = 从当前已覆盖技能 mask 出发，补齐剩余技能所需的最小人员下标元组。
```

边界：

```text
mask == full 时，不需要再选人，返回空元组。
```

转移：

```text
枚举每个人 i；
new_mask = mask | person_masks[i]；
如果 new_mask 比 mask 多覆盖了技能，就可以尝试选择这个人。
```

代码：

```python
class Solution:
    def smallestSufficientTeam(
        self,
        req_skills: List[str],
        people: List[List[str]],
    ) -> List[int]:
        skill_id = {s: i for i, s in enumerate(req_skills)}
        m = len(req_skills)
        full = (1 << m) - 1

        person_masks = []
        for skills in people:
            mask = 0
            for s in skills:
                mask |= 1 << skill_id[s]
            person_masks.append(mask)

        @cache
        def dfs(mask: int) -> tuple[int, ...]:
            if mask == full:
                return ()

            best = None

            for i, p in enumerate(person_masks):
                new_mask = mask | p
                if new_mask == mask:
                    continue

                candidate = (i,) + dfs(new_mask)
                if best is None or len(candidate) < len(best):
                    best = candidate

            return best

        return list(dfs(0))
```

这个写法的核心是：

```text
mask 是状态；
人是转移；
OR 用来合并技能集合。
```

注意：

```text
if new_mask == mask:
    continue
```

这句表示如果这个人不能提供任何新技能，那么从当前状态选他没有意义。

复盘：

```text
1125 = 位集合 + 最短团队 DP。
```

关键转化：

```text
不要把“选了哪些人”作为主要状态；
把“已经覆盖哪些技能”作为状态。
```

## AND/OR 单调性与 log trick

`log trick` 不是某一道题的专有名词，而是一类优化套路。

在 OR / AND / GCD 这类子数组题里，它指的是：

```text
虽然子数组有 O(n^2) 个；
但固定右端点时，所有以这个右端点结尾的子数组结果，去重后通常只有 O(log V) 种。
```

其中 `V` 是数值范围。对 32 位整数来说：

```text
log V 大约就是 30。
```

为什么成立：

```text
OR：扩展区间时，1 只会变多，不会变少。
AND：扩展区间时，1 只会变少，不会变多。
GCD：扩展区间时，值只会变成因子，变化次数有限。
```

统一模型：

```text
cur = 所有以当前位置结尾的子数组运算结果集合。
新元素 x 来了：
1. 新开一个子数组 [x]；
2. 之前所有以 i-1 结尾的子数组都接上 x。
```

模板：

```python
cur = set()
ans = set()

for x in nums:
    cur = {x} | {op(v, x) for v in cur}
    ans |= cur
```

对于不同运算：

```text
OR： op(v, x) = v | x
AND：op(v, x) = v & x
GCD：op(v, x) = gcd(v, x)
```

核心直觉：

```text
利用某些运算在区间扩展时的单调收敛性，
把每个右端点下的 O(n) 个子数组结果压缩成 O(log V) 个不同结果。
```

### [898. 子数组按位或操作](https://leetcode.cn/problems/bitwise-ors-of-subarrays/)

题目要求：

```text
所有连续子数组的 OR 结果有多少种。
```

注意它不是子集问题：

```text
子集：选或不选，可以不连续。
子数组：必须连续。
```

所以状态不能按“选或不选”组织，而要按右端点组织：

```text
cur = 所有以当前位置结尾的连续子数组 OR 值集合。
```

新来一个 `x`：

```text
1. 新开一个子数组 [x]，得到 x。
2. 把之前所有以 i-1 结尾的子数组接上 x，得到 v | x。
```

所以：

```python
cur = {x} | {v | x for v in cur}
```

完整代码：

```python
class Solution:
    def subarrayBitwiseORs(self, arr: List[int]) -> int:
        ans = set()
        cur = set()

        for x in arr:
            cur = {x} | {v | x for v in cur}
            ans |= cur

        return len(ans)
```

复杂度：

```text
每轮 cur 的大小约为 O(log V)，对 32 位整数约为 30。
总复杂度 O(n log V)。
```

复盘：

```text
898 = OR 单调性 + log trick。
```

关键转化：

```text
连续子数组问题，不要按选或不选想；
要按“以当前位置结尾”的状态想。
```

### [2411. 按位或最大的最小子数组长度](https://leetcode.cn/problems/smallest-subarrays-with-maximum-bitwise-or/)

题目对每个起点 `i` 问：

```text
从 i 开始的子数组，能达到的最大 OR 是多少？
达到这个最大 OR 的最短长度是多少？
```

对固定起点 `i`，能达到的最大 OR 是：

```text
nums[i] | nums[i+1] | ... | nums[n-1]
```

也就是后缀 OR，而不是整个数组 OR。

核心直觉：

```text
OR 的最大值由后缀里出现过哪些 1 位决定；
最短长度由这些 1 位最近在哪里出现决定。
```

倒着扫描，维护：

```text
last[bit] = 从当前位置 i 往右看，第 bit 位最近出现 1 的下标。
```

处理 `nums[i]` 时：

```text
1. 如果 nums[i] 的 bit 位是 1，那么 last[bit] = i。
2. 当前答案必须覆盖所有 last[bit] != -1 的位置。
3. 最短右端点 right = max(last[bit])。
4. ans[i] = right - i + 1。
```

例子：

```text
nums = [1, 0, 2, 1, 3]
```

从右往左：

```text
i = 4, nums[4] = 3 = 11
last[0] = 4
last[1] = 4
right = 4
ans[4] = 1
```

```text
i = 3, nums[3] = 1 = 01
last[0] = 3
last[1] = 4
right = 4
ans[3] = 2
```

```text
i = 2, nums[2] = 2 = 10
last[1] = 2
last[0] = 3
right = 3
ans[2] = 2
```

代码：

```python
class Solution:
    def smallestSubarrays(self, nums: List[int]) -> List[int]:
        n = len(nums)
        ans = [1] * n
        last = [-1] * 30

        for i in range(n - 1, -1, -1):
            x = nums[i]

            for bit in range(30):
                if (x >> bit) & 1:
                    last[bit] = i

            right = i
            for bit in range(30):
                if last[bit] != -1:
                    right = max(right, last[bit])

            ans[i] = right - i + 1

        return ans
```

复盘：

```text
2411 = OR 单调性 + 每一位最近出现位置。
```

关键转化：

```text
不是维护窗口内 OR；
而是倒着维护后缀中每个 bit 最近出现在哪里。
```
