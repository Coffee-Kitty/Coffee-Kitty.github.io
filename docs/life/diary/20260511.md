    workload -> Series.to_numpy -> na_value branch -> isna/_isna_array + values.copy
  - 它提交的 patch 也改在 pandas/core/base.py::to_numpy，和 human patch 的核心 edit layer 一致
  - benchmark 从 Mean: 0.1458804019 到 Mean: 1.398899e-05，提升非常明显
  - 两组 targeted pytest 都是 returncode 0，输出分别是 ... 和 ....

  它的 patch 思路是：当 na_value=np.nan、copy=False、底层是 numpy float/complex ndarray，并且 dtype 不需要转换时，把 na_value 当成默
  认值处理，从而跳过昂贵的 copy + isna mask。

  唯一小 caveat：它比 human patch 更保守，要求 dtype is None or np.dtype(dtype) == values.dtype，所以对当前 workload 很有效，但可能不
  会覆盖一些 human patch 能覆盖的 dtype conversion 场景。另外它包含了 complex dtype "c"，大概率合理，但如果追求最贴近 human patch，可
  以只保留 floating "f"。总体这个 case 可以算成功。
