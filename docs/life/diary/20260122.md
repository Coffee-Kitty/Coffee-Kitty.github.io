2. 关于cProfiler a+b 的时间算在那里的问题


原生类型（int/float）：cProfile 看不到 __iadd__，所有时间算在父函数头上。
```python
def heavy_calc():
    total = 0
    # 纯计算瓶颈
    for i in range(10_000_000):
        total += i  # 触发 __iadd__ 逻辑
    return total
```
虽然 Python 中“一切皆对象”，理论上 += 确实对应 __iadd__。但在 CPython 解释器层面，对于内置类型（int, float），加法操作被优化为 字节码指令 (Bytecode Instruction)，例如 INPLACE_ADD。 cProfile 监控的是 Function Call (栈帧的压入和弹出)。执行字节码指令不会产生新的栈帧。

TotTime 全部算在 heavy_calc 头上。



外部 C 扩展库（numpy）：cProfile 看得到 库函数调用，时间算在库函数头上。
```python
import numpy as np

def heavy_calc_numpy():
    arr = np.arange(10_000_000)
    # 调用外部 C 函数
    return np.sum(arr)
```
np.sum 是一个 Python 层面的函数对象（虽然它底层是 C）。cProfile 会记录 heavy_calc_numpy 调用了 numpy.sum（或者 numpy.core._methods._sum）。

TotTime 算在谁头上？
heavy_calc_numpy: TotTime ≈ 0。它只是发号施令。
numpy.sum (及其底层): TotTime ≈ 1.0s。脏活累活是它干的。



自定义对象：cProfile 看得到 __iadd__，时间算在 __iadd__ 头上。
```python
class MyInt:
    def __init__(self, val):
        self.val = val
    
    # 显式定义了 Python 级别的函数
    def __iadd__(self, other):
        self.val += other
        return self

def heavy_calc_custom():
    total = MyInt(0)
    for i in range(10_000_000):
        total += i  # 这里会触发函数调用！
    return total
```
TotTime 算在谁头上？
heavy_calc_custom: TotTime 很小（只有循环本身的开销）。
MyInt.__iadd__: TotTime 很大（累积值）。
注意：这里会有巨大的函数调用开销 (Function Call Overhead)。Python 调函数的成本很高，比加法本身高得多。
