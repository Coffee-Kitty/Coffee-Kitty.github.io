## inter-llms

该部分主要介绍 llm自我反馈，以及不涉及复杂工具的多llm合作与竞争。

1. 正确性 (Correctness)

在处理复杂任务时，多模型的集成与交互能够显著弥补单体模型的逻辑漏洞，因此评估集成系统的正确性至关重要。

这里我们将模型在多轮对话后自我反思（Self-reflection）也考虑在内。例如，LLF-Bench 评估了模型通过人类自然语言反馈进行持续学习与反思的能力；而 ReflectionBench 则对“反思”进行了深度解构，涵盖了新信息感知、记忆调用、信念更新、决策调整及元反思（对思考过程的反思）等多个认知维度。

另一种是多模型协作与竞争，可能涉及不同的模型，也可能是同一模型扮演多个关键角色。通过角色扮演和博弈，可以更全面地评估模型解决复杂问题的能力。AgentSims、CAMEL 与 LLMArena 利用角色扮演机制，在策略棋盘、纸牌游戏等场景中评估协作水平。MultiAgentBench 则在 6 个交互场景下，通过任务达成分、沟通分和规划分三个维度，对多智能体协作的正确性进行了系统化度量。

2. 鲁棒性 (Robustness)

在应用场景中，集成系统的鲁棒性不可忽视，比如抗干扰能力、面对内部冲突时的稳定性。多模型系统面临的独特挑战在于如何识别并处理“错误的共识”或“恶意节点的干扰”。TAMAS 证实了这点，TAMAS通过伪装、提示注入和合谋等手段，直接对协作过程进行对抗性鲁棒性测试，揭示了多智能体系统因复杂的通信链路及对“权威角色”的过度信任，可能暴露出比单体模型更脆弱的攻击面。

关于评估工作，MAS-Resilience 通过在系统中注入“故障模型”，测试集成系统是否仍能通过交互机制保持结论的一致性。GOVSIM 向协作群体中注入“自利型新来者”，实验发现这种干扰会导致最佳模型的生存率从 $53.3\%$ 剧降至 $33.3\%$。

3. 安全性 (Safety)

多个 LLM 的协作可能诱发新兴的安全风险。评估集成系统的安全性，不仅要防御外部攻击，更要警惕智能体之间可能产生的“有毒协作”。X-Teaming 揭示了攻击者如何利用多智能体协作链条，将一条恶意指令拆解为多个看似无害的片段进行传播，从而绕过单体模型的安全屏障。

在复杂的社交场景中，SOTOPIA 提出了更高维度的要求：模型必须在追求个人目标的同时，处理竞争性利益冲突，并严格遵守社会规范、保守机密。

此情境下目前还缺乏专门的评估框架。

4. 效率 (Efficiency)

多模型集成虽能提升性能，但其负面影响是计算资源与 Token 消耗的急剧增加。因此，在评估时必须权衡协作带来的收益与产生的开销。

MultiAgentBench 在评估Correctness的同时引入了里程碑达成率（Milestone Achievement Rates）与交互轮数等指标，用以量化协作的“进度”是否高效。
而在社会化交互中，SOTOPIA 发现了一个有趣的现象：在达成谈判或妥协等目标时，人类平均每轮仅需约 $16.8$ 个词，而 GPT-4 则需约 $45.5$ 个词。这表明目前的 AI 智能体在社交互动效率上与人类仍有较大差距，高频率、高密度的信息交换成为了集成系统亟待优化的方向。专门的评估基准尚待研究。


3.2.3 多大语言模型集成测试 (中文版)
多大语言模型（Multi-LLMs）集成是指多个模型（或同一模型的多个副本）通过协作、竞争或自我反馈机制共同完成任务。这种集成方式能够通过角色扮演和多轮交互处理超越单体模型能力的复杂任务。

正确性 (Correctness) 是集成测试的首要指标。多模型交互能有效弥补单体模型的逻辑漏洞。评估维度包括：
自我反思（Self-reflection）：模型在多轮对话中根据反馈进行自我修正的能力。例如，LLF-Bench 评估模型从自然语言反馈中学习的能力；ReflectionBench 则细化了新信息感知、信念更新等认知反思维度。
协作与竞争：通过角色扮演或博弈评估解决问题的全面性。AgentSims、CAMEL 和 LLMArena 利用角色扮演测试策略博弈能力；MultiAgentBench 则通过任务达成、沟通和规划三个维度对多智能体协作进行系统化度量。

鲁棒性 (Robustness) 关注系统在面对内部冲突、干扰或恶意节点时的稳定性。多模型系统因复杂的通信链路及对“权威角色”的信任，可能比单体模型更脆弱。TAMAS 通过伪装和提示注入测试对抗鲁棒性；MAS-Resilience 通过注入“故障模型”测试结论一致性；GOVSIM 则揭示了“自利型新来者”会导致协作群体的生存率大幅下降。

安全性 (Safety) 在多模型环境下引入了“有毒协作”等新风险。X-Teaming 发现攻击者可将恶意指令拆分为无害片段分发给不同模型，从而绕过单体防御。SOTOPIA 则要求模型在竞争性社交场景中平衡个人目标、社会规范与机密保守。目前该领域仍缺乏专门的系统性安全评估框架。

效率 (Efficiency) 是实际部署的关键。多模型集成显著增加了计算资源和 Token 消耗。MultiAgentBench 引入了里程碑达成率与交互轮数来量化进度。SOTOPIA 发现 GPT-4 在社交互动中的词耗远高于人类，表明高频冗余的信息交换是集成系统亟待优化的瓶颈。专门的系统性评估基准尚待研究。
