

## 理解coverage的三个文件
### 理解run_to_run_isolation
把原始 workload 脚本改写成"每次 timeit 迭代在独立子进程中执行"的版本。
这样可以消除以下 run-to-run 干扰因素：
- 模块级缓存（lru_cache, dict cache 等）
- 全局状态污染
- Python 解释器热身效应
- JIT 编译器的 warmup（如 numba）
- 内存碎片化
